# 脚本局部说明

本文件适用于 `scripts/`。同时继承仓库根目录的规则。

- 先按脚本职责限定调查范围：构建、smoke、迁移、媒体处理或安全检查，不要预读全部脚本。
- 修改脚本时优先运行同名或直接相关的 `*.test.mjs` / `*.test.ts`，再根据调用方决定是否构建对应 package。
- 自托管构建、原子发布或线上 smoke 任务还需读取 `deploy/AGENTS.md` 和相关部署 runbook。
- 数据迁移、回填、清理和恢复脚本可能产生不可逆写入；默认只做静态检查或 dry-run。执行真实数据写入前必须确认目标环境、备份与用户授权。
- 脚本日志不得输出环境文件内容、密钥、密码、完整会话令牌或私钥。
