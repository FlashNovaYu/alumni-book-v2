# Phase 9: 慢网与低端设备手动性能验收清单

本清单旨在指导开发人员和测试人员模拟真实网络（Slow 4G）及低端设备场景，确保本阶段性能优化措施在真实浏览器中完全发挥效用。

## 1. 模拟环境配置

在 Chrome / Edge 等 Chromium 内核浏览器中：
- **网络模拟 (Network Throttling)**: 选择 `Slow 4G` (或手动配置：Latency: 2000ms, Download: 400Kb/s, Upload: 150Kb/s)。
- **CPU 降速 (CPU Throttling)**: 选择 `4x slowdown`。
- **设备视口**: 调整为 iPhone 12/13/14 Pro 尺寸 (375px × 812px)。
- **动效减弱**: 在开发者工具中输入 Ctrl+Shift+P，搜索并选择 `Emulate CSS media feature prefers-reduced-motion: reduce`，然后在 no-preference 状态下各测试一遍。

## 2. 页面专项验收清单

| 页面 | 关键检测点 | 预期合格指标 | 验收结果 |
|---|---|---|---|
| **首页 (/)** | 1. 首页输入框展现时间 <br>2. 提交按钮可用性 <br>3. 静态字体渲染 | 1. 慢网加载时，输入框在 1s 内立即可见，无需等待任何外部 JS <br>2. 输入名字并点击提交时反应即时，请求 verify 轻量接口 <br>3. 无字体闪烁或布局重排 (FOUT) | `[ ]` 待测 |
| **同学录 (/roster)** | 1. 卡片入场闪烁防范 <br>2. 搜索过滤体验 <br>3. 头像加载成本 | 1. 进入页面时卡片按 CSS 渐显入场，禁止大面积黑屏或隐藏等待 <br>2. 输入名字搜索时，卡片仅根据过滤状态隐藏/显示，不重播整体淡入动画，输入极度流畅不卡顿 <br>3. 默认请求 160px 的轻量头像，不加载大图原图 | `[ ]` 待测 |
| **学生详情页 (/student/:slug)** | 1. 页面内容首屏可见性 <br>2. SWR 更新闪动防范 <br>3. 音乐与编辑按钮布局 | 1. 慢网下首先直出静态内容，不能有超过 2 秒的白屏/隐藏等待 <br>2. 挂载后，SWR 后台同步数据，数据若改变，页面只局部更新，不重播全部淡入动画 <br>3. 375px 小屏下，“分享 TA”、“编辑资料”和“音乐播放器”等悬浮按钮不重叠 | `[ ]` 待测 |
| **相册页 (/album)** | 1. 灯箱大图预览 <br>2. 缩略图体积 | 1. 灯箱支持键盘左右键和 Esc 关闭，小屏下灯箱按钮不遮挡图片 <br>2. 网格只加载 480 宽的缩略图，仅在打开灯箱时加载高清原图 | `[ ]` 待测 |
| **时光轴 (/timeline)** | 1. 滚动性能 <br>2. 过滤切换 | 1. 时光轴整体为纯 CSS 效果，网络上不加载 ScrollTrigger <br>2. 点击年份/标签时响应在 100ms 内，无卡顿 | `[ ]` 待测 |
| **年度册 (/yearbook)** | 1. 图片懒加载 <br>2. 打印版式 | 1. 往下滚动时才按需加载年度册大图 <br>2. Ctrl+P 调出打印预览时版式规整，无重叠或多余的 JS 浮标 | `[ ]` 待测 |

## 3. 问题反馈与跟踪

如果在验收中发现任何不符合上述指标的行为（例如慢网下某元素被长时间隐藏），请记录下：
1. **所属页面和具体组件**
2. **重现时的网络及 CPU 设定**
3. **报错信息或表现证据** (例如截屏或 network 中加载的 chunk 文件名)
