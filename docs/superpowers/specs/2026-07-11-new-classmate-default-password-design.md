# 新增同学默认账号初始化设计

日期：2026-07-11

## 目标

管理员在后台新增同学时，系统自动创建可登录的同学账号。初始密码固定为 `123456`，同学首次登录后必须自行修改密码。

## 服务端行为

- `POST /api/students` 不接收或返回明文密码。
- 创建时使用既有 `hashPassword` 将 `123456` 写入 `students.account_password_hash`。
- 同时写入 `account_initial_password_changed = 0` 与 `account_status = 'pending'`。
- 已有的登录接口继续把该状态返回为 `mustChangePassword: true`；首次成功改密后由既有逻辑设为已激活。
- 管理员通过编辑页重置初始密码的既有行为不变。

## 后台反馈

- 新建学生弹窗不新增密码输入项。
- 创建成功后，后台明确显示“已创建同学账号，初始密码为 123456，请通知同学首次登录后修改”。
- 列表中的状态继续显示为“待改密”。

## 验收与回归

1. Worker 测试创建学生后直接查询账号字段，确认密码为 PBKDF2 哈希、首次改密标记为 `0`、状态为 `pending`。
2. 使用创建学生的 slug 与 `123456` 调用登录接口，确认返回 200 与 `mustChangePassword: true`。
3. 后台静态契约测试确认创建成功提示包含初始密码说明。
4. 既有管理员初始密码重置测试保持通过。

## 非目标

- 不为本次新增“自定义初始密码”表单、批量导入账号或邮件通知功能。
- 不修改已有同学的账号密码。
