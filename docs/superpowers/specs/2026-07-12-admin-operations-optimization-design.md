# 管理后台运营体验优化设计

日期：2026-07-12

## 目标

完成管理后台的七项运营体验优化：历史公共投稿可删除、相册拖放上传、完整时光轴管理、管理员学生预览、可读操作日志、符合先发后审模式的工作台，以及精简且实际生效的站点设置。

成功标准：每一项操作都有现有权限模型约束；所有删除与重要写入继续保留审计；不把管理令牌暴露到 URL；不引入新的拖拽或表单依赖。

## 范围与非目标

本次只调整后台管理体验和必要的公开站点会话、配置消费逻辑。

- 不新增内容回收站、版本管理、工作流引擎或新的管理员角色。
- 不改变公共群聊“先发后审”的发布策略。
- 不更改相册照片既有的压缩、R2 上传与封面机制。
- 不通过像素位置推导时光轴日期；日期始终由明确表单字段保存。

## 页面设计

### 审核中心与工作台

历史公共投稿页签中的每张记录都提供删除入口，不再仅允许待审核记录通过或退回。删除前必须填写原因；成功后从当前列表移除，并写入既有审计日志。

公共群聊目前先发布再治理，不会产生可处理的待审核条目。工作台保留“个人留言待审核”待办，并将“公共留言待审核”替换为运营概览中的“今日公共群聊”计数，跳转至审核中心的公共群聊页签。统计范围是当天创建、`client_nonce` 为非历史投稿标识的公共群聊消息，而不是 `public_messages` 的 `pending` 记录。

### 相册上传

恢复提交 `05282921e6804fbcc4af809b5b73bf1e923ae7ac` 中曾实现的拖放上传区：

- 点击区域打开原生多文件选择器；
- 文件拖入时显示高亮状态，放下后复用同一个上传函数；
- 继续逐张压缩并上传，不改变现有 R2 接口；
- 上传中禁用关闭按钮，发生错误时保留清晰错误反馈。

### 时光轴管理

时光轴后台使用“完整事件列表 + 编辑抽屉”的模型。

- 管理专用读取接口只返回 `timeline_events` 的全部事件，按 `event_date DESC, sort_order ASC` 排序，不混入照片、留言或同学加入等派生项目，也不施加公开时间流的 100 条限制。
- 事件按日期分组显示。每个卡片的拖拽手柄仅能在同一个日期组内调整展示顺序。
- 拖拽完成时提交该日期组的完整 ID 顺序，后端验证每个事件存在且日期相同，按数组位置更新已有 `sort_order`，并写入一条排序审计记录。
- 点击卡片打开编辑抽屉，维护日期、标题、描述、照片 R2 Key、事件类型和里程碑。日期变更后事件进入新日期组；该编辑继续使用原有更新接口及审计机制。

### 学生页面预览

学生列表的“预览”链接仍直接打开公开学生页。公开站点的登录守卫在没有同学会话时，读取同源 `admin_token` 并调用现有管理员验证接口。仅令牌有效时允许管理员继续查看页面；验证失败仍按原逻辑跳转首页。

管理员令牌不会追加到 URL、不会写入同学会话、不会赋予新的 API 权限。预览仅绕过公开站点的前端登录重定向。

### 操作日志

操作日志页面不再渲染原始动作代码、资源 ID 或 JSON `<pre>` 块。新增一个前端摘要格式化器：

- 将动作映射为自然语言，例如“更新相册”“上传照片”“删除时光轴事件”；
- 从变更前后摘要提取可读的资源名称和字段标签；
- 将字段差异显示为简洁的“修改了：标题、封面照片”等信息；
- 删除、隐藏、撤回等操作额外显示填写的原因；
- 旧日志缺少名称或结构不符合预期时，降级为“完成了一项相册管理操作”等中文描述，不显示内部 ID 或代码。

审计数据结构与保留策略不变。

### 站点设置

在既有前言、页脚、字体、纪念馆和致谢设置之外，只增加“站点基本资料”卡片：

- 站点名称；
- 班级名称与届别；
- 分享摘要。

`SiteConfig` 新增带默认值的 `identity` 对象，配置接口同样返回默认值以兼容旧数据。站点名称用于浏览器标题，班级信息用于页脚，分享摘要用于页面描述元信息。客户端会在配置保存后的页面访问中同步标题与页脚；静态 HTML 的分享元信息在下一次站点构建时生成更新。

## 接口、权限与数据

| 能力 | 接口/数据 | 权限 | 审计 |
| --- | --- | --- | --- |
| 删除历史公共投稿 | 复用现有管理员删除接口，补齐后台调用 | `moderation.manage` | 保留删除原因与原记录摘要 |
| 拖放相册上传 | 复用 `/api/upload` | `content.manage` | 保持既有文件上传审计 |
| 完整时光轴与排序 | 管理读取与排序接口；复用 `timeline_events.sort_order` | `content.manage` | 创建、编辑、排序、删除均记录 |
| 管理员预览 | 复用管理员令牌验证接口 | 有效管理员会话 | 无写操作，不记录 |
| 工作台群聊统计 | 工作台聚合查询 | `moderation.view` | 无写操作，不记录 |
| 站点基本资料 | 扩展 `SiteConfig.identity` | 主管理员（保持现有设置写入限制） | 复用站点设置更新审计 |

所有写接口继续由 Worker 的会话与权限中间件保护。前端隐藏按钮不作为权限保障。

## 错误处理与兼容性

- 拖放区域忽略空文件集；上传、压缩和网络错误均显示失败提示，并结束加载状态。
- 排序接口拒绝空数组、重复 ID、跨日期 ID 或不存在的事件，不更改任何排序。
- 时光轴编辑仍要求标题和日期；网络失败保留抽屉内输入，方便重试。
- 管理员预览的验证请求异常或返回未授权时一律回到原始登录守卫流程。
- 新增 `identity` 缺失时使用稳定的默认文案，旧站点配置无需迁移。
- 审计摘要解析异常只影响该条显示，不影响整个日志页面加载。

## 验证策略

1. Worker 测试：验证公共投稿删除审计、时光轴完整读取与同日排序校验、工作台的公共群聊日统计、站点基本资料默认值与持久化。
2. 静态页面测试：验证历史投稿删除按钮、相册拖放上传、时光轴拖拽与编辑抽屉、管理员预览验证、日志中不存在原始 JSON 渲染、工作台不再声明公共留言待审核，以及设置字段。
3. 构建验证：运行 Worker 测试、后台测试/类型检查/构建，以及公开站点类型检查与测试。
4. 手动冒烟：在管理员登录态依次验证上传拖放、时光轴排序与编辑、学生预览、公共群聊统计、删除原因和操作日志可读性。

## 实施顺序

1. 先补 Worker 数据与接口测试，确保排序、统计和配置默认值可验证。
2. 实现后台各页面交互与操作日志摘要。
3. 调整公开站点守卫、标题和页脚的配置消费。
4. 执行全量自动验证并进行管理员端手动冒烟。
