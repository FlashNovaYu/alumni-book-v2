# 公开站点界面反馈修复设计规格

日期：2026-07-12

## 背景与已确认决策

本轮依据人物长廊与班级空间的浏览器批注，处理可复现的显示问题，并收敛站点的信息架构。用户已确认采用推荐方向：保持复古纸本主题，以书签、细线、印章和简约 SVG 图形取代表情符号及不明所以的切换控件。

需要解决的具体问题：

1. 顶部导航同时暴露“班级空间”“影像馆”“时光轴”，与班级空间的聚合定位重复；导航状态条从右向左切换时视觉方向错误。
2. 人物长廊缺少分页；历史中 `0528292` 曾实现分页，但回归修复时删除。每页必须恰好显示 9 位同学。
3. 个别头像资源失效时会露出破碎图片；既有首字降级必须继续可靠生效。
4. 公开站点仍有硬编码装饰性 emoji。
5. 班级空间的三个文字加计数锚点像调试列表；时间轴把普通留言、照片上传、新成员加入都自动写入“班级大事”，且照片 URL 被重复拼接，导致缩略图加载失败。

## 范围与非目标

### 范围

- 顶部导航保留“前言、同学档案、班级空间、年度册、更多”，其中“更多”进入新增的 `/more` 占位页；影像馆和时光轴保留独立 URL，并从班级空间访问。
- 顶部导航将使用纸张书签底与墨线，按实际移动方向动画；桌面和移动抽屉的目录保持同一信息架构。
- 人物长廊按 9 张卡片分页，搜索条件变化时回到第 1 页，页码按钮可键盘访问并带当前页语义。
- 所有由本仓库代码提供的装饰性 emoji 改成已有或新增的简约 SVG 图标；用户输入、表情反应数据不改写。
- 班级空间章节导航改为具有章节编号、名称、说明与计数的“目录卡”；窄屏保持横向可滚动而非挤压。
- 班级空间的“班级大事”只读取管理员手工维护的 `timeline_events`。独立的完整时光轴继续保留既有综合浏览能力；照片应由“精选影像”展示，需要在班级大事展示的照片由管理员作为事件配图选择。
- 班级空间时间轴最多显示最新 6 条手工事件，正确解析相对和绝对文件 URL，并以连续时间线而非宽大的横向空卡片呈现。

### 非目标

- 不修改 R2、D1 中已有历史数据，也不自动删除已经存在的时间轴记录。
- 不移除独立的 `/album`、`/timeline`、`/yearbook` 页面或管理后台的时光轴 CRUD。
- 不更改用户输入的留言文本、头像、相册内容或反应表情的存储格式。
- 不部署、推送或合并到 `main`。

## 方案比较

### 方案 A：仅隐藏重复导航，保留现有时间轴聚合逻辑

改动最小，但仍会把普通变动错误地称为“班级大事”，图片 URL 仍可能重复拼接，也无法解决卡片留白。

### 方案 B：目录式导航 + 人工策展时间轴（采用）

顶部导航只保留高频入口；班级空间成为影像和时间轴的入口。时间轴默认只查询 `timeline_events`，管理员通过现有 CRUD 精确决定何时、何事值得记录。前端把预览改为紧凑的连续事件轨。

这符合“同学录目录”和“班级大事”的语义，不引入新表或新工作流，也能直接修复图片 URL。

### 方案 C：增加多个自动事件开关

能让管理员选择是否记录上传、加入、留言，但需要额外配置、迁移、后台表单和解释成本；对当前需求过度设计。

## 设计细节

### 导航与更多页

`TopNav.astro` 的单一 `navItems` 数组成为桌面和移动抽屉的唯一来源，顺序固定为前言、同学档案、班级空间、年度册、更多。`/more` 是普通纸张占位页，仅显示“新的章节正在整理，敬请期待”。影像馆与时光轴仍可由班级空间进入；当前页样式在 `/more` 生效。

`navRuntime.ts` 持久记录上一次活动项的水平位置；更新书签和墨线时，根据目标位于左侧还是右侧设置 `data-nav-direction`。CSS 为向左移动使用右侧缩放原点，为向右移动使用左侧缩放原点。首次测量不播放过渡，避免页面交换时从左边错误展开。

### 人物长廊与头像

`RosterWall.vue` 恢复可分页的派生列表，`PAGE_SIZE` 为 9。页码在总页数不大于 9 时完整显示，超过时保持首尾页和当前页邻近页，并以省略号收束。切页后焦点或滚动回到检索区；搜索关键词和动态名单更新都校正当前页。

`ArchiveRosterCard.vue` 保留当前 `@error` 首字降级；分页测试使用坏 URL 来证明卡片不会显示破图。

### 无 emoji 的图形语言

只替换仓库代码中的装饰性 emoji：年度册标题与打印按钮、账号中心状态、榜单标题、音乐播放器和分享按钮。使用语义化文本、细线分隔和现有 `components/icons` 中的 SVG；缺少的图形新增小型、单色、可设置 `size` 的 Astro SVG 组件。互动反应仍使用既有数据值，因为它是用户内容和服务端协议的一部分，不属于装饰。

### 班级空间

章节导航通过 `sections` 传入 `index`、`label`、`description` 和 `count`，桌面显示为窄书页目录，活动条固定在左侧。移动端显示为横向目录卡片，仍使用 `IntersectionObserver` 反映阅读位置。

Worker 的 `getTimelineFeed` 新增显式 `curatedOnly` 选项。班级空间概览传入 `curatedOnly: true`，仅查询手工 `timeline_events`；独立 `/api/timeline` 保持既有综合浏览能力。这样普通留言、相册照片、学生注册不会再自动出现在“班级大事”，但不会移除完整时光轴已有数据。班级空间概览把预览数从 8 减至 6，`counts.timelineItems` 反映实际预览数量。

`ClassSpaceTimelineRail.vue` 使用纵向连续时间线：日期、类型、标题、简短说明和可选配图放在收紧的纸卡中。其图片解析器接受以下三种格式：完整 HTTPS URL、`/api/files/<key>` 相对 API URL、裸 R2 key；第二种不会重复附加 `/api/files/`。

## 验收标准

1. 档案页在 10 名以上数据时首屏仅有 9 张 `archive-card`，页码切换后显示下一批；关键词搜索始终从第 1 页开始。
2. 顶部与移动导航都不再将影像馆、时光轴列作一级项目，并都有“更多”；`/more` 可直接打开。
3. 所有仓库硬编码的装饰 emoji 均改为 SVG 或文字；反应协议不变。
4. 班级空间概览只返回手工时间轴事件，独立时光轴仍可获得其既有综合数据；预览含 `/api/files/...` 图片时请求路径恰为一个 `/api/files/` 前缀。
5. 班级空间在 1433px 和 390px 宽度均不横向溢出，目录和时间线的可读性符合纸本主题。
6. `pnpm --filter site-astro typecheck`、针对性的 Vitest、Worker Vitest 与 Playwright 用例通过；全量验证若受已有 `JWT_SECRET` 或线上构建数据依赖阻断，必须在结果中单独报告。
