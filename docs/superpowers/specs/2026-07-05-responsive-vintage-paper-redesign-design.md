# 同学录响应式复古纸张改版设计规格

日期：2026-07-05

## 1. 背景

当前公开站点已经具备同学账号登录、人物长廊、个人主页、相册、时间轴、年度册、留言互动等功能，但视觉和排版仍有几个核心问题：

- 首页登录界面在桌面端重心偏右，标题与登录书本距离过大，阅读路径割裂。
- 首页作为登录入口时不需要承载大量站内信息，但需要更强的入馆仪式感。
- 手机端与电脑端目前更多是“压缩同一套布局”，缺少各自适配的结构。
- 多处页面同时存在淡入、GSAP、图片加载、局部水合和视图过渡，容易造成重复加载、跳变和卡顿。
- 当前视觉混合了纪念馆、书本、贴纸、深色封面等元素，缺少一套统一的站内设计系统。

本规格定义下一阶段的整体视觉和响应式排版方向。它不是实现计划，后续需要再拆成实施步骤。

## 2. 目标

1. 首页改为“同学录封面开场 → 下滑出现登录按钮 → 点击进入登录信息环节”的两段式登录体验。
2. 全站公开页面统一升级为“复古纸张同学录”风格。
3. 为桌面端和手机端分别定义排版骨架，而不是简单缩放。
4. 保留同学录的温度、纸张、档案、照片、印章、贴纸趣味，同时避免幼稚和过度装饰。
5. 明确动画和加载约束，减少重复加载、跳变、卡顿。
6. 让后续实现可以按页面逐步落地，并能通过视觉、性能、交互三类验收。

## 3. 非目标

- 不把首页改成站点目录页或营销页。首页只负责登录入口和入馆仪式。
- 不在本阶段新增后端数据模型。
- 不重做后台管理端，除非公开站点样式令牌需要同步少量兼容变量。
- 不新增复杂 3D、全景空间或重型动画库作为首页核心。
- 不为了装饰牺牲首屏性能和手机流畅度。

## 4. 设计方向总览

整体体验分成两个视觉层级：

1. 首页封面：暗场、高级、仪式感强。它像“翻开同学录前的一瞬间”，使用深色背景、大标题、低透明纸页/照片/印章暗纹，用户向下滑动后看到“翻开同学录”按钮。
2. 站内页面：复古纸张系统。进入登录表单和站内页面后，界面切换到米白纸张、棕色控件、细线分区、低饱和卡片、档案和书页语言。

这两个层级需要共享同一套主题来源：同学录、纸张、青春档案、照片背面、班级印章。暗场首页不能变成科技风，站内纸张也不能变成普通后台仪表盘。

## 5. 首页设计

### 5.1 信息结构

首页只包含三段：

1. 封面首屏
   - 小标签：同学录 / CLASS MEMORY MUSEUM / 入馆封面。
   - 大标题：围绕“翻开青春、遇见同窗、青春纪念馆”等主题。
   - 一句短副文案，长度控制在 1-2 行。
   - 向下滑动提示。

2. 登录按钮段
   - 用户下滑后，出现主按钮“翻开同学录，进入登录”。
   - 按钮点击后平滑滚动到登录信息区。
   - 该段不放人物、相册、时间轴等站内入口。

3. 登录信息区
   - 账号输入或姓名输入。
   - 密码输入。
   - 登录按钮。
   - 首次登录说明。
   - 错误提示和加载状态。
   - 首次登录改密弹窗仍保留，但视觉需要适配纸张风格。

### 5.2 桌面端布局

- 首屏高度使用 `100svh`，大标题居中偏上，不再采用左右两栏。
- 顶部导航在首页应弱化：
  - 未登录时只展示品牌或简化导航，不展示站内功能入口，避免误导。
  - 登录后站内页面再展示完整导航。
- 暗场背景可以使用：
  - 低透明纸条、胶片、照片轮廓。
  - 低透明大号中文暗纹，例如“翻开我们的青春”。
  - 细网格或纸纤维噪声，但不能显得像科技 SaaS。
- 登录按钮段在首屏后方出现，桌面端按钮宽度约 320-380px。
- 登录信息区采用宽版纸页卡片：
  - 左侧是简短说明。
  - 右侧是表单。
  - 最大宽度控制在 860-980px。

### 5.3 手机端布局

- 首屏仍保留暗场封面，但标题需要缩短并自动换行，避免超出屏幕。
- 向下滑动提示和主按钮要进入拇指可触范围。
- 登录按钮可以在用户下滑到一定位置后靠近底部安全区出现。
- 登录信息区必须单列：
  - 标题。
  - 首次登录提示。
  - 账号输入。
  - 密码输入。
  - 登录按钮。
- 不使用双页书本，不在手机端保留中间书脊。

### 5.4 首页动效

首页只允许一条主线动效：

- 首屏加载：标题、标签、副文案轻微错位淡入。
- 下滑：封面内容轻微上移，登录按钮淡入。
- 点击按钮：平滑滚动到登录信息区，表单轻微浮现。

约束：

- 不叠加多个全局 reveal、GSAP 入场和 Vue 动画。
- 不在首屏加载重型图片或远程字体。
- `prefers-reduced-motion: reduce` 下必须关闭位移和滚动联动，只保留可见状态。
- 移动端不使用视差。

## 6. 复古纸张设计系统

### 6.1 视觉关键词

- 米白纸面。
- 旧书页边界。
- 棕色主控件。
- 印章红危险/强调。
- 低饱和卡片。
- 细分隔线。
- 档案夹、书页、照片背面、便签和贴纸。
- 安静、温暖、整洁、有年代感。

### 6.2 颜色

建议令牌方向：

- 主棕：用于主按钮、选中态、滑杆、重点标签。
- 次棕：用于 hover、浅按钮、纸页边。
- 米白背景：全站 canvas。
- 卡片白：主要内容卡片。
- 纸边色：边框、分割线。
- 印章红：删除、错误、重要提醒。
- 柔和金：小标签、装饰线、纪念馆感。
- 墨色：正文和标题。

颜色使用上避免：

- 大面积纯黑，除首页封面外。
- 大面积紫蓝渐变。
- 饱和橙棕堆满页面。
- 每个卡片都不同颜色。

### 6.3 字体与字号

- 标题可继续使用衬线字体栈，保留旧书和纪念册气质。
- 正文使用系统无衬线，保证中文阅读清晰。
- 不使用负字距。
- 不使用随视口宽度缩放的字体。
- 首页大标题可独立设置响应式断点，但不能用 `vw` 造成过窄或过宽时失控。

### 6.4 圆角与边框

- 常规按钮、输入框、标签：4-8px。
- 页面卡片：8-14px。
- 胶囊只用于导航、标签、滑块、少量状态组件。
- 主要分区使用细边框和纸色差异，不依赖大阴影。

### 6.5 组件风格

按钮：

- 主按钮：棕色填充，白色文字。
- 次按钮：浅纸色背景，棕色或墨色文字。
- 危险按钮：印章红。
- 禁用态：低对比纸边色，不能只靠透明度。

输入框：

- 米白或卡片白背景。
- 细边框。
- 聚焦态使用主棕描边和轻微阴影。
- 手机端高度不低于 44px。

卡片：

- 纸面背景。
- 细边框。
- 少量阴影。
- 卡片内部使用标题、说明、状态区，不堆过多装饰。

标签：

- 低饱和底色。
- 控制数量和换行。
- 手机端避免挤成多行墙。

## 7. 全站响应式排版

### 7.1 全局结构

建立统一页面骨架：

- `page-shell`：控制页面背景、顶部留白和最小高度。
- `page-header`：页面标题区，桌面居中或左右结构，手机紧凑。
- `content-grid`：页面主体网格。
- `paper-panel`：纸张面板。
- `archive-card`：档案卡片。
- `section-divider`：纸页分区线。

页面顶部留白应统一考虑固定导航高度，避免每个页面自行写 `calc(var(--nav-height) + var(--spacing-section))` 导致不一致。

### 7.2 导航

桌面端：

- 保留悬浮导航，但改为纸张胶囊风格。
- 宽度可以固定，但需要根据登录态和导航项数量保证不挤压。
- 首页未登录状态只保留品牌或极简入口。

手机端：

- 导航折叠为顶部纸条。
- 展开菜单要像纸页抽屉，而不是普通透明玻璃层。
- 已登录身份栏需要在菜单底部稳定显示，避免闪烁。

### 7.3 前言页

目标：像翻开同学录后的第一页。

桌面端：

- 标题居中。
- 正文放在窄纸页中，最大宽度约 720px。
- 特别致谢用小头像卡片或签名区。

手机端：

- 正文行高略高，但左右边距收紧。
- 底部操作按钮改为纵向全宽。

### 7.4 人物长廊

目标：从“卡片网格”升级为“档案检索墙”。

桌面端：

- 搜索区像档案夹条，居中或左对齐均可，但需要稳定。
- 人物卡 2-3 列，根据宽度自适应。
- 卡片包含头像、姓名、短座右铭、标签、资料状态。
- 排行榜面板与人物列表视觉统一，不能像独立插件。
- 班级图谱和座位记忆作为底部亮点卡，不抢主列表。

手机端：

- 搜索区置顶在页面标题下方。
- 人物卡建议单列，若信息足够少可以双列头像卡，但不能压缩座右铭。
- 排行榜改成横向分段卡或折叠区。

### 7.5 个人主页

目标：像一份完整的个人档案，而不是一长串普通信息表。

桌面端：

- 顶部改为“档案封面”：
  - 头像、姓名、昵称、座右铭、资料完整度、标签。
  - 背景图降低透明度并纸张化。
- 主体采用两栏或分区：
  - 左侧可放身份摘要、访问数、自助编辑入口。
  - 右侧放详细资料分区。
- 基础信息、联系方式、个性标签、兴趣馆藏、校园回忆、时间胶囊要有统一 section 样式。
- 照片墙、留言墙、亮点功能作为后续分区，不与基础资料混在一起。

手机端：

- 顶部紧凑显示头像、姓名、资料完整度。
- 各资料分区单列，标题短、间距稳定。
- 自助编辑和分享按钮不能遮挡正文，优先放到内容内或底部安全区。
- 留言贴纸在手机端取消旋转和 hover 放大。

### 7.6 相册页

目标：纸张相册，不是普通图片网格。

桌面端：

- 相册分组有纸页标题和标签。
- 图片网格可采用 3 列，保持稳定 aspect-ratio。
- frame-retro、frame-film、frame-polaroid 需要统一边距和阴影，不应造成高度跳动。

手机端：

- 图片 2 列。
- 标签筛选横向滚动或自动换行，但不能撑高首屏太多。
- 灯箱按钮需要更大触控面积。

### 7.7 时间轴

目标：纸页年表。

桌面端：

- 可保留左右交错，但卡片宽度、线条和节点要更纸张化。
- 筛选 Tab 改成纸张分段控件。

手机端：

- 强制单侧时间轴。
- 节点和卡片间距固定，避免切换筛选时大跳。
- 图片加载需预留尺寸。

### 7.8 年度册

目标：在线预览和打印导出都保持纸张风格。

- 屏幕预览使用纸张卡片和章节分隔。
- 打印样式继续保留干净白底，不能被深色首页风格污染。
- 手机端年度册可以保持简化，不作为主要移动阅读场景。

### 7.9 留言墙

目标：保留贴纸趣味，但提升稳定性。

桌面端：

- 贴纸样式继续保留复古胶带、黑板、拍立得、横格便签。
- 列表可以有轻微错落，但旋转角度要控制。
- 表情反应和主人回复要有清晰层级。

手机端：

- 默认不旋转留言卡。
- hover 动效改为无位移或轻微背景变化。
- 贴纸样式选择器需要改成可横向滚动的 segmented control。

## 8. 动画与性能约束

### 8.1 动画治理原则

- 每个页面只允许一个主入场系统。
- 全局 reveal、页面淡入、组件自身淡入不能重复作用在同一元素上。
- 优先 CSS transition 和 opacity/transform。
- GSAP 只用于确有必要的滚动联动或复杂序列。
- 移动端禁用视差和重型 stagger。

### 8.2 加载稳定性

需要重点处理：

- 首页封面到登录区的滚动动效不能影响布局高度。
- 图片必须有 `width/height` 或稳定 `aspect-ratio`。
- 懒加载组件 anchor 需要预留高度，避免出现后顶动页面。
- SWR 数据刷新不能重置用户正在交互的表单。
- 视图过渡后不得重复绑定滚动监听。
- 导航登录态显示要避免先隐藏后出现的闪烁。

### 8.3 减少网络和水合压力

- 首页首屏不加载站内数据。
- 首页只加载登录所需 JS。
- 非首屏组件继续延迟加载，但需要稳定占位。
- 相册、留言、图谱、座位记忆保持可见后加载。
- 复古纸张纹理优先 CSS 实现，不依赖大图。

## 9. 可访问性

- 所有按钮和输入框保留清晰 focus-visible。
- 暗场首页文字对比度必须满足可读性。
- 手机端触控目标不小于 44px。
- 登录错误信息需要文本提示，不能只靠颜色。
- 动画遵守 `prefers-reduced-motion`。
- 页面结构使用合理标题层级。

## 10. 验收标准

### 10.1 视觉验收

- 桌面首页首屏不再左右割裂，标题在视觉中心。
- 首页首屏不出现账号密码表单，用户下滑后才看到登录按钮。
- 点击“翻开同学录”能平滑进入登录信息区。
- 站内页面统一呈现复古纸张风格，而不是混合纪念馆、玻璃、普通卡片和科技黑场。
- 手机端不是桌面端的简单压缩，每个核心页面都有适配排版。

### 10.2 交互验收

- 首页登录流程在桌面和手机均能完成。
- 首次登录改密仍能正常触发。
- 页面切换后导航登录态稳定。
- 人物搜索、相册筛选、时间轴筛选、留言提交和回复不受样式改版影响。

### 10.3 性能验收

- 首页首屏无明显跳变。
- 页面加载动画不重复播放。
- 切换页面时不出现主内容闪烁两次。
- 手机端滚动无明显卡顿。
- Playwright 基础登录和页面访问测试通过。
- 站点构建、类型检查和现有测试通过。

## 11. 建议实施顺序

后续实施计划建议拆成 5 个阶段：

1. 设计令牌和全局页面骨架。
2. 首页封面登录流程。
3. 导航、前言、人物长廊响应式改版。
4. 个人主页、留言墙、相册、时间轴响应式改版。
5. 动画去重、性能验收、移动端视觉回归。

每个阶段都需要独立验证，避免一次性大改导致定位困难。
