# 移动端陀螺仪光影、主题水波与两列档案墙设计规格

## 1. 背景与目标

当前公开站点已经拥有纸页档案视觉、明暗主题和卡片倾斜基础，但移动端仍有四类未完成问题：

1. 阿里云预发布入口为 HTTP，浏览器处于非安全上下文时不会提供 `DeviceOrientationEvent`，现有“开启 3D 光影”按钮会把失败误报为成功。
2. 方向事件只更新卡片旋转，档案卡和相册卡的光影层仍以鼠标悬停作为唯一显示条件，手机倾斜时光斑不可见。
3. `MainLayout.astro` 的全局样式存在重复/错位片段，移动环境光媒体查询没有稳定进入构建结果。
4. 手机端档案墙必须保持两列；当前卡片、导航按钮、分页按钮、时间轴筛选条及账号页存在尺寸、边距或滚动适配不足。
5. 明暗切换水波必须从实际明暗图标中心开始，不能依赖固定页面坐标或假定按钮内部结构。

本规格的目标是：在不改变现有纸页视觉语言、站点路由、认证和 API 的前提下，完成移动端交互和布局修整，并为 HTTP 环境提供诚实、可用的降级体验。

## 2. 非目标与保护项

### 保护项

- 保留现有 `paper` 与 `night` 两套主题令牌和持久化键 `alumni_theme`。
- 保留现有顶部导航信息架构、公开站点路由、登录门控和页面内容。
- 保留桌面端鼠标倾斜交互及其现有卡片静态旋转构图。
- 保留现有主题水波的 View Transition 实现，不引入新的动画库。

### 非目标

- 不调整 Worker、D1、R2、认证接口或数据结构。
- 不在本任务中强行把阿里云 HTTP 入口改造成 HTTPS；代码需要检测并提示该限制。
- 不重做整站品牌、字体或配色。
- 不把手机端档案墙改成单列；两列是明确的产品约束。

## 3. 方案与决策

采用“共享陀螺仪状态 + 卡片光影统一联动”的方案。

### 方案 A：共享交互状态（采用）

- 扩展 `useMouseTilt.ts` 的共享状态，统一处理权限、方向事件、平滑插值、光影坐标和能力状态。
- `ArchiveRosterCard.vue` 与 `AlbumGrid.vue` 消费相同的旋转/光影状态。
- 通过安全上下文与权限探测返回 `granted`、`unsupported`、`insecure-context`、`denied` 等明确状态。
- HTTP 或权限拒绝时使用触摸/指针位置驱动的轻量光影，不显示虚假的“已开启陀螺仪”。

优点是逻辑集中、事件监听不会重复、档案与相册行为一致，且能在后续 HTTPS 发布后自动启用真实传感器。

### 方案 B：组件内独立监听

档案卡和相册各自申请权限、监听事件并维护动画状态。隔离性较强，但会重复权限流程和 RAF 生命周期，容易产生两个传感器状态，不采用。

### 方案 C：只做触摸光影

完全不接入设备方向，只用触摸/指针模拟。实现最简单，但无法满足真实陀螺仪目标，不采用。

## 4. 交互设计

### 4.1 传感器能力与用户反馈

`initDeviceOrientation()` 改为返回明确结果，不再无条件把按钮标记为成功：

```ts
type DeviceOrientationStatus =
  | 'granted'
  | 'unsupported'
  | 'insecure-context'
  | 'denied'
  | 'error'
```

- 非浏览器环境直接返回 `unsupported`。
- `window.isSecureContext === false` 时返回 `insecure-context`，不调用权限 API。
- iOS 等需要手势授权的平台只在按钮点击中申请权限。
- 权限成功后才启动一个共享 `deviceorientation` 监听器和一个 RAF 平滑循环。
- 权限拒绝、HTTP 或事件不可用时，按钮保留可重新尝试的状态，并显示简短说明；不隐藏为“已开启”。
- 页面销毁时清除监听器、RAF 和当前组件注册，避免跨文档导航残留。

### 4.2 旋转与光影映射

- 以首次有效方向事件作为基准，`beta` 和 `gamma` 经限制后映射为卡片 `rotateX`、`rotateY`。
- 同一方向值同时映射为光影中心 `glareX`、`glareY`，保证倾斜方向和光斑方向一致。
- 传感器激活时，光影层显示条件为“方向交互激活或触摸交互激活”；桌面端仍以 hover 为主要显示条件。
- 方向平滑仅修改 `transform`、CSS 自定义属性和 `opacity`，不触发尺寸布局。
- `prefers-reduced-motion: reduce` 时禁用 RAF 和方向动画，保持静态卡片与静态光影。

### 4.3 HTTP/权限降级

- 触摸/指针移动时计算卡片内部归一化坐标，驱动同一组 `glareX`、`glareY`。
- 在不安全上下文或权限拒绝时，按钮文案反映当前状态，例如“使用触摸光影”或“HTTPS 后开启陀螺仪”。
- 降级状态不改变卡片点击、分页和相册灯箱行为。

### 4.4 明暗主题水波

- 主题运行时从实际显示的 `[data-theme-toggle]` 按钮中找到可见按钮，再查找其图标元素。
- 动画原点使用图标 `getBoundingClientRect()` 的中心；找不到图标时回退到按钮中心。
- 坐标使用视口 CSS 像素，半径覆盖当前视口四角。
- 两个导航主题按钮继续同步 `aria-pressed`、`aria-label` 和持久化主题。
- 减少动态效果时立即切换主题，不创建水波动画。

## 5. 移动端布局设计

### 5.1 两列档案墙

- 在 375–430px 常见手机视口保持两列，不使用 `max-width: 400px` 直接切换为单列。
- 使用两列 `minmax(0, 1fr)`，卡片内容必须允许收缩。
- 移动卡片采用紧凑但可读的内边距、头像和行距；姓名、座右铭和状态使用截断或行数限制，不撑破网格。
- 卡片之间保留稳定间距，页面不得出现横向滚动。
- 3D 光影按钮和分页控件不改变两列结构。

### 5.2 触控尺寸

- 顶部菜单、音效、明暗主题、关闭抽屉等图标按钮统一不小于 44×44px。
- “开启 3D 光影”、分页按钮、时间轴筛选、信箱标签和学生页操作按钮统一提供至少 44px 的触控高度；视觉图标可小于 44px，但点击区域不能小于 44px。
- 输入控件在移动端保持至少 44px 高度和 16px 字号，避免 iOS 自动缩放。

### 5.3 页面级适配

- 清理 `MainLayout.astro` 中重复/错位的全局样式，只保留一份环境光、尘埃和 reduced-motion 规则。
- 修正环境光移动媒体查询，确保移动端光斑尺寸和位置真正生效。
- 时间轴筛选条保留横向滚动能力，但隐藏原生滚动条并增加可见的边缘提示/滚动阴影，不产生页面级横向溢出。
- 账号页标题、页面容器和卡片在小屏使用统一水平内边距，不贴边裁切。
- 学生页操作按钮在小屏自动换行或形成等宽行，保持文字和对比度可读。

## 7. TDD 与验证策略

先写失败测试，再写最小实现：

1. 方向状态测试：HTTP、不支持、拒绝、授权成功分别返回正确状态；授权失败不会进入“已开启”。
2. 光影状态测试：方向状态能改变旋转和光影坐标；未交互时光影不会错误显示；触摸降级能驱动光斑。
3. 主题动画测试：可见移动主题按钮的图标中心被用于 `clip-path` 起点；无图标时回退按钮中心；减少动态效果不创建动画。
4. 静态布局测试：移动 CSS 保持两列档案墙、44px 触控区域、统一小屏边距，并确认布局样式没有重复错位片段。
5. 浏览器测试：390×844 与 430×932 视口检查两列、无横向溢出、主题切换、传感器降级状态、时间轴滚动和账号页边距。
6. 完成后运行目标测试，再运行 `pnpm --filter site-astro typecheck`、站点测试与适配的完整验证门禁。

## 8. 验收标准

- [ ] 390px、430px 视口均显示两列同学卡片。
- [ ] 档案墙、时间轴、学生页、账号页和顶部导航无页面级横向溢出。
- [ ] 真实 HTTPS + 授权成功时，档案卡和相册卡的旋转与光斑同步变化。
- [ ] HTTP、非支持浏览器或权限拒绝时，不显示虚假成功状态，并有可用降级光影。
- [ ] 所有移动端主要操作控件触控区域至少 44px。
- [ ] 明暗水波从实际图标中心展开，两个主题按钮状态保持同步。
- [ ] 减少动态效果下无设备方向 RAF、无水波动画、页面内容仍完整可读。
- [ ] 现有桌面鼠标倾斜、主题持久化、路由和认证行为无回归。
- [ ] 规格中无未解决占位符、范围外接口变更或未确认的视觉重做。

## 9. 设计自审记录

- 范围聚焦在一个移动端体验问题组，未扩展到后端或品牌重做。
- HTTP 限制被显式建模为能力状态和降级路径，不把部署条件误报为前端成功。
- 两列档案墙已作为硬约束写入布局与验收，而不是沿用单列兜底。
- 主题水波保留现有 View Transition 架构，只替换起点测量逻辑。
- 所有动画均包含 reduced-motion 终态，所有新增行为均有对应测试入口。
