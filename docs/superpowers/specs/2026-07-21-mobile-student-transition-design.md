# 移动端同学档案卡过渡节奏设计

## 背景

当前同学档案卡进入个人页、以及个人页返回档案页的边缘扩散动画使用固定桌面端节奏：边缘动画 0.82 秒，头像与姓名共享元素移动 0.7 秒。手机屏幕和卡片的相对尺寸不同，扩散更早覆盖可视区域，导致动画主观上“一瞬间完成”。

## 目标

- 仅优化触摸设备/窄屏（`max-width: 768px`）的档案卡进入和返回动画。
- 手机端让扩散、坍缩和头像/姓名移动都能被清楚感知，同时保持两个环节的时间对齐。
- 桌面端现有时长、关键帧、层级和视觉结果完全不变。
- 保留 `prefers-reduced-motion` 的无动画降级行为。

## 方案

在 `packages/site-astro/src/styles/view-transitions.css` 增加移动端媒体查询，仅覆盖档案卡相关 View Transition 伪元素：

1. 进入和返回的边缘动画总时长调整到约 1.05 秒。
2. 移动端关键帧增加较舒缓的起步阶段，避免一开始就快速越过卡片边缘；中段持续扩散，最后阶段完成全屏覆盖或回收。
3. 头像/姓名共享元素在扩散开始后的约 0.14 秒开始移动，移动时长约 0.9 秒，使其在边界覆盖完成前逐步到位。
4. 卡片详情坍缩同步放慢到约 0.62 秒，继续保持在新页面背景层之上。
5. 返回动画使用同一组时长和相反方向，不增加新的 JavaScript 状态分支。

## 非目标

- 不修改桌面端动画参数。
- 不改动 View Transition 状态管理、路由逻辑、身份元素命名或页面结构。
- 不新增基于设备性能的动态计算。

## 验收标准

- 移动端进入和返回动画不再瞬间完成，扩散/坍缩过程可感知且连续。
- 头像和姓名从扩散早期开始移动，并在扩散完成时落位，不出现旧快照重影。
- 桌面端现有档案卡进入/返回测试全部保持通过。
- 新增静态测试锁定移动端媒体查询中的时长、延迟和返回动画规则。
- 类型检查、相关单元测试和移动视口 Playwright 测试通过。
