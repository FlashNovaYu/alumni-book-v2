# 同学录下一阶段外观与功能升级设计规格

日期: 2026-06-27

## 1. 设计结论

下一阶段采用混合方向:

- 以 **A 青春纪念馆** 作为整体高级感基底。
- 吸收 **B 互动手账本** 的贴纸、粒子、留言、签章和照片角标等局部温度。
- 谨慎加入 **C 班级宇宙档案馆** 的班级图谱、馆藏统计、座位/全景空间作为亮点功能。

目标不是把网站做成花哨皮肤，而是把现有同学录升级为一套稳定、统一、可运营、可扩展的数字纪念册体验。所有新增动画和视觉效果都必须服从 Phase 9 已建立的性能边界: 内容先可见，动效后增强，低端设备可降级，不允许再次引入全站 GSAP/ScrollTrigger 预加载。

## 2. 体验定位

### 2.1 情绪关键词

- 高级、克制、有仪式感。
- 温暖、亲密、有同学录纸本记忆。
- 有少量惊喜，但不科技炫技。

### 2.2 视觉语言

主视觉采用“纪念馆 + 年度册”系统:

- 基底色: 深墨灰、校史金、米白纸张、轻胶片蓝。
- 材质感: 纸张纹理、相纸边、档案夹、展柜线框、签章。
- 排版: 大标题有仪式感，内容区信息密度适中，卡片圆角控制在 8px 内。
- 图片: 真实头像、相册和班级照片优先，装饰图形只做辅助。

局部互动采用“手账”元素:

- 贴纸留言、祝福便签、照片角标、纪念签章、轻量粒子。
- 手账元素只出现在留言、照片、个人页局部模块中，不能占据全站主视觉。

亮点功能采用“档案馆数据层”:

- 班级图谱、馆藏统计、年度成就、座位/教室空间入口。
- 仅在用户主动进入或滚动到视口后加载，不影响常规浏览。

## 3. 页面级设计

### 3.1 首页: 入馆仪式

首页从“输入姓名进入”升级为“打开纪念馆”的第一幕。

保留姓名门禁，但视觉上加入:

- 展馆式 Hero: 大标题、班级年份、纪念馆副标题。
- 胶片/相册暗纹背景，首屏可轻微漂移。
- 姓名输入框做成“访客登记卡”。
- 验证成功后不做大幅炫光跳转，使用轻量翻页或展门打开动效进入前言/同学录。

成功标准:

- 首页首屏 JS 仍保持极轻，不为了 Hero 动画引入重型库。
- 输入框在慢网下立即可用，不能等待背景或粒子加载。

### 3.2 前言页: 序厅

前言页承担“为什么这本同学录值得打开”的情绪铺垫。

改造方向:

- 由普通文本墙升级为序厅式分屏: 班级寄语、老师/班委寄语、年度关键词。
- 支持后台配置序厅文案、署名和配图。
- 可加入 3 到 5 个缓慢浮动的时间碎片，例如“第一次运动会”“毕业照”“晚自习”。

### 3.3 同学录页: 人物长廊

同学录页从普通网格升级为人物长廊。

核心改造:

- 保留搜索与筛选，但视觉上变为“档案检索”。
- 每张同学卡片增加头像、昵称、座右铭、标签、资料完成度。
- 支持按回忆标签、兴趣标签、宿舍/小组/座位分组。
- 排行榜不再抢首屏，可变为“馆藏热度”小面板。
- 无主页或资料未完善的同学也要有清晰状态，不再像功能缺失。

性能要求:

- 列表筛选不能触发大规模重复入场动画。
- 卡片出现使用 CSS Stagger 或局部 transition，不能引入全局动画控制。

### 3.4 个人页: 档案展柜

个人页是下一阶段的核心体验。

布局改造:

- 顶部 Hero 变为个人展柜: 头像、姓名、昵称、签名、背景图、个人签章。
- 基础信息、兴趣爱好、校园回忆、未来规划不再是相似表格堆叠，改成“档案分区”。
- 照片墙以相册胶片和相纸形式出现。
- 留言墙变为祝福贴纸墙，可选卡片样式、表情反应、主人回复。
- 自助编辑入口更明确，展示当前资料完整度和缺失字段。

可新增模块:

- “我的关键词”: 从兴趣、回忆、未来字段生成标签云。
- “我的时间胶囊”: 给未来自己的话或毕业寄语重点呈现。
- “同桌/好友关联”: 与班级图谱打通，但初期可先做静态字段或后台选择。

### 3.5 相册页: 影像馆

相册页从图片列表升级为影像馆。

改造方向:

- 按活动、年份、地点、人物标签分组。
- 支持精选照片作为展馆入口大图。
- 照片打开后展示说明、拍摄时间、关联同学。
- 后台可维护照片标签和排序。

动画要求:

- 缩略图使用懒加载和渐显。
- 大图查看器可用轻量 CSS/原生交互，不默认加载重动画库。

### 3.6 时间轴页: 校史走廊

时间轴页作为全站叙事骨架。

改造方向:

- 后台维护事件类型: 班级大事、活动、考试、毕业节点、趣事。
- 前台按时间展示，并允许从事件跳转到相册、同学、留言。
- 时间轴节点可有轻微展柜开合效果。

### 3.7 年度册页: 成册导出

年度册页作为“把网站变成一本书”的出口。

改造方向:

- 页面结构与纪念馆主题统一。
- 支持选择公开数据生成年度册。
- 导出前展示缺失项提示，例如头像缺失、寄语缺失、照片不足。

## 4. 新增功能范围

### 4.1 第一阶段必须完成

- 统一视觉主题 token: 色彩、字体、阴影、边框、卡片、纸张/展柜样式。
- 首页、同学录页、个人页三大核心页面改造。
- 留言墙手账化: 卡片样式、贴纸感、置顶、主人回复展示优化。
- 个人资料完整度和缺失字段提示。
- 后台表单与前台字段映射补齐，确保已有字段能显示、能编辑、能保存。
- 动画 owner 规则固化: 每个选择器只能由一个系统控制。

### 4.2 第二阶段应完成

- 相册影像馆改造，支持标签、排序、精选入口。
- 时间轴校史走廊改造，后台可维护事件与关联资源。
- 个人页新增关键词、时间胶囊、资料完成度模块。
- 管理后台增加“前台呈现预览”和“缺失内容巡检”。

### 4.3 第三阶段亮点功能

- 班级图谱: 以兴趣、座位、小组、留言互动、同桌关系为节点和边。
- 座位/教室全景: 可先做 2D 座位图，后续再考虑全景照片。
- 馆藏统计: 浏览、留言、照片、资料完整度、活跃同学等数据面板。
- 年度成就: 自动生成“最受欢迎照片”“留言最多档案”“共同关键词”等。

第三阶段功能不能阻塞第一阶段上线。若数据不足，必须展示空状态和引导补全，而不是隐藏功能入口造成“功能不存在”的错觉。

## 5. 后台与数据联通设计

### 5.1 管理后台要补齐的能力

- 学生管理: 所有前台展示字段都能编辑，包括兴趣、校园回忆、未来规划、个人小传、隐私可见度。
- 相册管理: 支持标签、排序、精选、关联同学。
- 留言审核: 支持置顶、隐藏、主人回复、卡片样式审核。
- 时间轴管理: 支持事件类型、日期、封面图、关联相册/学生。
- 站点设置: 支持首页文案、序厅文案、主题开关、粒子强度、是否启用图谱/座位图等。
- 内容巡检: 展示头像缺失、资料缺失、无照片、无留言、字段未公开等问题。

### 5.2 API 与数据要求

优先复用现有 D1 schema 和 `students.info` JSON 字段，不在第一阶段拆大表。

需要明确的 API 能力:

- 公开读取: 学生列表、学生详情、相册、时间轴、排行榜、站点配置。
- 管理写入: 学生字段、相册标签排序、留言状态与回复、时间轴事件、站点配置。
- 自助编辑: 当前登录同学只能编辑自己的允许字段，并遵循隐私设置。

所有新增字段必须同步更新:

- `packages/shared` 类型定义。
- Worker 序列化/反序列化。
- 前台展示组件。
- 后台编辑表单。
- 静态测试或 API 测试。

## 6. 动画与性能边界

### 6.1 允许的动画

- CSS 首屏入场、纸张轻微浮动、卡片 hover、照片渐显、签章轻压。
- IntersectionObserver 触发的轻量 reveal。
- 仅在用户进入视口或主动打开功能时，按需加载图谱/全景/照片查看器。

### 6.2 禁止的动画

- 全站默认加载 GSAP 或 ScrollTrigger。
- 用 `.fade-in` 这类通用 class 让多个系统争夺同一元素。
- 为等待动画而先隐藏正文内容。
- 列表筛选、SWR 刷新、搜索输入时重播大规模动画。
- 移动端默认开启背景视差或大量粒子。

### 6.3 降级策略

- `prefers-reduced-motion` 下所有装饰动画关闭。
- 移动端关闭视差和大粒子，只保留基本 transition。
- 慢网或 JS 失败时，核心内容直接可见。
- 图谱/全景失败时显示解释性空状态和重试入口。

## 7. 组件边界

建议新增或重组以下组件:

- `MuseumHero`: 首页入馆 Hero，不依赖重动画。
- `VisitorPass`: 姓名门禁登记卡。
- `ArchiveRosterCard`: 人物长廊卡片。
- `ProfileCompleteness`: 资料完整度与缺失项提示。
- `MemoryStickerWall`: 留言贴纸墙，对接现有 `MessageWall` 数据。
- `GalleryExhibitGrid`: 影像馆照片网格。
- `TimelineCorridor`: 校史走廊时间轴。
- `ClassGraphPreview`: 班级图谱入口，懒加载完整图谱。
- `ContentAuditPanel`: 后台内容巡检面板。

组件设计原则:

- 前台视觉组件只接收已整理好的 view model，避免在模板里散落字段规则。
- 后台编辑组件与前台展示共享类型定义。
- 动画逻辑不得跨组件用全局选择器抓取节点。

## 8. 验收标准

### 8.1 体验验收

- 首页、同学录、个人页看起来属于同一套“青春纪念馆”系统。
- 手账元素有温度但不喧宾夺主。
- 访客能清楚理解每个功能入口，不再出现“功能做了但看不到”的问题。
- 资料缺失时有清晰空状态和补全路径。

### 8.2 数据联通验收

- 前台展示字段均能在后台或自助编辑中找到对应入口。
- 后台保存后，刷新前台能看到结果。
- 留言、相册、时间轴、排行榜与前台页面状态一致。
- 权限与隐私设置不会泄露隐藏字段。

### 8.3 性能验收

- `pnpm verify:all` 通过。
- `node scripts/perf-budget.mjs` 通过，并继续确认首页、同学录、时间轴首屏不加载 GSAP/ScrollTrigger。
- 关键页面在慢网模拟下正文先可见，没有跳变、重复加载或长时间空白。
- 对动画重的图谱/全景功能增加懒加载或网络级测试。

### 8.4 回归验收

- 姓名门禁、管理员登录、学生编辑、留言审核、相册管理、时间轴管理均保持可用。
- R2 图片/音乐资源 URL 拼接仍正确。
- 自定义 HTML 专属模板不被新主题破坏。

## 9. 分阶段实施建议

### Phase 10.1: 视觉系统与核心页面

完成主题 token、首页、同学录、个人页、留言墙视觉升级，并保持现有 API 不大改。

### Phase 10.2: 后台联通与内容巡检

补齐后台字段、站点配置、内容巡检、前台空状态和编辑入口。

### Phase 10.3: 影像馆与校史走廊

升级相册与时间轴，加入资源关联和后台管理能力。

### Phase 10.4: 图谱与全景亮点

加入班级图谱、座位/教室空间、馆藏统计。全部按需加载，不进入首屏关键路径。

## 10. 非目标

- 不在第一阶段重写整个数据库模型。
- 不把后台改造成全新产品，只做与前台体验相关的管理能力补齐。
- 不追求 3D 全站化或大量粒子特效。
- 不牺牲 Phase 9 已完成的性能优化成果。
- 不默认改动专属 `customHtml` iframe 模板的内部样式。
