# 档案墙分页动效增强设计

## 背景

当前档案墙已改为整页容器的双向分页动画，并通过稳定最小高度消除了分页时的纵向抽动。但固定 `30px` 横向位移在手机和桌面端都过于克制，缺少旧版页面切换中清晰、利落的飞入飞出感。

历史实现提供了两组可复用的动效特征：

- 旧档案墙单向入场使用 `scale(0.95)`、透明度变化和指数型缓动。
- 旧页面切换使用离场 `0.34s`、入场 `0.4s` 与 `cubic-bezier(0.22, 1, 0.36, 1)`，入场略慢于离场。

## 目标

- 明显增强分页的横向移动幅度，同时避免接近整屏滑动的生硬感。
- 页码增大和减小时使用严格镜像的飞入飞出方向。
- 保留当前整页切换、稳定高度、移动端两列和搜索高度重算行为。
- 尊重系统的“减少动态效果”偏好。

## 非目标

- 不恢复逐卡片 `TransitionGroup` 动画。
- 不添加 GSAP 或其他动画依赖。
- 不改变分页数量、排序、搜索或卡片内部陀螺仪光影逻辑。
- 不修改全站跨文档页面转场。

## 动效参数

分页容器使用响应式横向位移：

```css
--roster-page-shift: clamp(96px, 22vw, 280px);
```

- 入场：`0.4s cubic-bezier(0.22, 1, 0.36, 1)`。
- 离场：`0.34s cubic-bezier(0.22, 1, 0.36, 1)`。
- 初始或结束状态：`opacity: 0` 与 `scale(0.95)`。
- 稳态：浏览器默认的 `opacity: 1` 与 `transform: none`。

该位移在 390px 手机端为 96px，在常见桌面端随视口增加，并在 280px 封顶。

## 双向规则

| 操作 | 当前页离场 | 新页入场 |
| --- | --- | --- |
| 前进到更大页码 | 向左移动一个位移量 | 从右侧一个位移量进入 |
| 后退到更小页码 | 向右移动一个位移量 | 从左侧一个位移量进入 |

前进与后退只改变 X 轴符号，持续时间、缓动、缩放和透明度完全一致。

## 布局与性能

- 动画继续作用于 `.roster-page` 整页容器，卡片自身的倾斜和光影 `transform` 不参与分页动画。
- 离场页继续绝对定位并覆盖在同一稳定视口内。
- `.roster-page-viewport` 的已测最大高度继续保留，防止最后一页卡片较少时文档高度塌陷。
- 仅过渡 `opacity` 与 `transform`，并保留 `will-change`，不触发布局属性动画。

## 无障碍降级

现有全局 `prefers-reduced-motion: reduce` 规则继续将动画压缩为近乎即时切换，不增加额外覆盖规则。

## 验收标准

- 390px 移动端仍为两列，无横向溢出。
- 前进分页的离场 X 位移为负、入场 X 位移为正，关键帧幅度为 96px。
- 后退分页的离场 X 位移为正、入场 X 位移为负，关键帧幅度为 96px。
- 桌面端关键帧位移随视口增大且不超过 280px。
- 从 12 张卡片切换到仅 1 张卡片时，动画期间和结束后的分页视口高度不下降。
- 前后翻页的 `window.scrollY` 变化不超过 2px。
- 档案墙分页、移动端陀螺仪与主要响应式视觉测试继续通过。
