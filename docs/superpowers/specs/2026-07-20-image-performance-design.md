# 同学录图片性能与上传压缩设计说明书

## 目标

在不改变现有页面功能、旧图片 URL 和访问权限的前提下，降低相册、同学头像、学生背景图、时间线图片、班级空间图片以及私聊/群聊头像的首屏流量、解码开销和布局抖动；所有新上传图片都经过适量压缩并生成适合展示尺寸的派生图。

## 当前基线与约束

- 项目使用 Astro/Vue 前端、Hono Worker、D1 元数据和 R2 文件存储。
- `packages/shared/src/imageUtils.ts` 已提供 `compressImage`、`generateImageVariants` 和变体表单封装。
- `workers/api/src/routes/upload.ts` 和 `workers/api/src/routes/classmate.ts` 已能校验并保存 `media_json` 变体元数据，但同学自助上传前端尚未提交变体。
- 旧记录可能只有原始 R2 key 或旧 JSON 字符串数组，必须继续可读。
- 当前工作区已有 `MessageWall.vue`、`TopNav.astro`、`workers/api/src/db/schema.sql` 和 `workers/api/migrations/0020_student_checkins.sql` 的用户改动，不得覆盖或重构。
- 本阶段不迁移 R2 数据，不要求阿里云账号或密钥，不新增生产基础设施。

## 方案

### 上传处理

统一管理员上传和同学自助上传的浏览器预处理：

| 资源 | 原图最大展示处理宽度 | 派生宽度 | 用途 |
| --- | ---: | --- | --- |
| 头像 | 400px | 128/256/320px | 列表、卡片、个人页 |
| 背景图 | 1600px | 320/960px | 移动端、桌面端 Hero |
| 相册照片 | 1600px | 320/960px | 网格、照片墙、灯箱邻图预加载 |

`compressImage` 的输出规则：

- SVG 不做栅格化，保持现有行为。
- GIF 不强行转码，保持动画和兼容性。
- 有透明度的 PNG 优先尝试 WebP；WebP 编码不可用时回退 PNG。
- 无透明度的 JPEG/PNG 优先尝试 WebP；编码失败时回退 JPEG/PNG。
- 只有生成结果确实小于输入时才替换原文件，否则保留输入文件，避免“压缩”反而增大体积。
- 派生图质量保持在现有 `0.82` 附近，原图仍保留作为高清灯箱和旧模板兜底。

管理员和自助上传均提交：`file`、`type`、必要的 `slug/albumId`、`variants` 元数据以及对应的 `variant_<kind>` 文件。Worker 继续执行格式、尺寸、变体数量、路径前缀和总大小校验，并在 D1 写入失败时删除本次上传对象。

### 媒体 URL 组装

继续使用 `packages/shared/src/media.ts` 的 `buildMediaSources` 作为唯一组装入口。组件只提供原始 URL、变体列表、展示宽高；禁止新增局部的 `/api/files/` 拼接逻辑。

当变体不存在时，函数继续返回原始 URL；当变体存在时，返回按宽度排序的 `srcset`、匹配展示区域的 `sizes`、最小变体作为默认 `src`，并保留原始 URL 供高清预览使用。

### 页面展示策略

- 相册网格、照片墙、名录卡片、管理员缩略图：`srcset + sizes + loading="lazy" + decoding="async" + width/height`。
- 当前学生页 Hero 头像：保留首屏 eager，但使用头像变体和稳定尺寸。
- 学生背景图：使用学生媒体元数据的 320/960 变体；没有变体时回退原图。背景仍由 CSS 控制，不改变叠加渐变、裁剪和定位。
- 灯箱主图：打开后加载原图，增加 `decoding="async"` 和错误兜底；相邻图片预加载使用 960 变体，避免提前下载原图。
- 时间线、班级空间相册封面、私聊/群聊头像、排行榜、收件人选择器、致谢头像和编辑预览：统一补齐合适的变体/尺寸属性；没有媒体元数据的接口继续使用原 URL。
- 不为纯 CSS 装饰图和 SVG data URI 引入网络请求。

### 阿里云兼容边界

本阶段将媒体源解析和 Worker 文件服务保持为可替换边界：

- 当前仍使用 R2 和 `/api/files/<key>`，不执行数据迁移。
- 未来接入阿里云 OSS/CDN 时，只替换媒体 URL 解析或文件服务适配层，不修改各个 Vue/Astro 图片组件。
- 不把阿里云 AccessKey、服务器地址或部署脚本写入仓库。
- 若未来使用阿里云服务器做批量回填，使用独立脚本读取现有 R2/D1 元数据，不能成为在线上传的强依赖。

## 验证与验收

1. shared：图片变体、压缩输出、旧 URL 和无变体回退测试通过。
2. Worker：管理员与同学自助上传的变体校验、D1 元数据写入、失败补偿和旧对象清理测试通过。
3. site/admin：静态检查确认所有目标图片入口使用响应式属性；关键组件类型检查和构建通过。
4. 浏览器性能：相册/照片墙列表不请求原图；灯箱仅在打开时请求原图；背景图和头像优先命中变体。
5. 兼容性：旧数据库记录、旧 R2 URL、透明 PNG、GIF、上传取消和错误回滚行为不变。
6. 运行 `pnpm verify:worker`、`pnpm verify:shared`、针对性的 admin/site 测试，最后按风险运行 `pnpm verify:all`。

## 非目标

- 本阶段不迁移现有 R2 对象到阿里云。
- 不引入 Cloudflare Images、阿里云 IMM 或新的在线动态转码服务。
- 不重做图片 UI、灯箱交互或页面视觉风格。
