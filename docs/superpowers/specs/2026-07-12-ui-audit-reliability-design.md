# UI 审计问题修复设计

## 目标

修复公开站点和管理后台已复现的显示、交互和无障碍问题，同时为会话失效、异常上传与异常内容建立可验证的降级行为。修复不得修改线上既有数据。

## 范围

### 会话与错误状态

- 前台同学接口收到 401 时统一清理 `classmate_account_token`、同学资料和姓名缓存。
- 页面显示“登录已失效，请重新登录”的反馈，并跳回首页；信箱和班级空间不再分别展示空数据或无效的重试按钮。
- 保留正常网络错误的页面内错误状态，不把网络波动误判为登录失效。

### 上传与头像降级

- 管理端上传和同学自助上传都验证声明 MIME、扩展名和文件签名的一致性，只接受 JPEG、PNG、WebP、GIF。
- 服务器使用验证后的扩展名和 MIME 写入 R2，拒绝将 HEIC 内容伪装为 PNG 的文件。
- 头像展示组件在图片加载失败时切换到姓名首字；年度册与后台学生列表补齐现有缺失的降级。
- 已经存储的无效 HEIC 对象不在本次代码改动中迁移；管理员重新上传有效图片即可恢复头像。

### 页面稳定性与内容边界

- 学生页在 mounted 后读取浏览器会话，避免 SSR 和 hydration 产生不同 DOM。
- 年度册为头像添加显式尺寸约束，并加大顶部安全间距，避免 CLS 与固定导航遮挡打印按钮。
- 时光轴、年度册和后台仪表盘对超长留言、巡检项使用有限行展示与总数说明，避免一条异常内容占据整页。

### 导航和无障碍

- 修正控制台快捷入口 `/config` 为已有的 `/settings`。
- “更多功能”下拉菜单支持鼠标、焦点、Enter、Escape 与 `aria-expanded` 状态同步。
- 为搜索、留言、设置页控件补足可访问名称；移动端后台图标导航补充 `aria-label`/`title`。
- 将公开站点移动端菜单按钮与关闭按钮提升到至少 44px 触控目标。

### 验证链路

- `site-astro` 的组合测试命令先构建后台产物，再运行需要后台 `dist` 的导航测试。
- 为会话失效、上传格式验证、头像降级、下拉菜单状态、错误路由和内容截断分别添加最小回归测试。
- 完成后执行 Worker、Site、Admin 类型检查/测试/构建，并用 Chromium 复测公开站点与后台的桌面和手机视口。

## 不在范围内

- 不直接修改生产 D1 或 R2 中已有学生资料、头像对象和留言数据。
- 不重新设计整体视觉语言、重构未涉及的页面，或引入新的图片转码服务。
- 不增加新的身份认证方式或改变已有权限模型。

## 实施顺序

1. 先为每项可自动化行为编写失败测试，确认现状可复现。
2. 完成 Worker 上传校验与前台会话失效的统一处理。
3. 完成头像、hydration、内容边界和导航/表单的局部 UI 修复。
4. 修复组合测试脚本，运行全量验证与浏览器回归。

## 风险与决策

- Worker 运行时没有通用图片转码能力，因此采用“拒绝不受支持编码 + 前端稳定降级”，而不是在上传时转码 HEIC。
- 会话失效只由明确 401 触发，避免将临时网络失败导致的 5xx/连接错误错误登出。
- 内容截断保留完整内容在管理端详情页处理；列表和年度册优先保证页面可浏览。
