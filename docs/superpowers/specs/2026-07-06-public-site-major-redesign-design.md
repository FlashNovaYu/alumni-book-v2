# 同学录公开站点大幅重构设计规格

日期：2026-07-06

## 1. 背景

当前公开站点已经完成了一轮响应式复古纸张改版，首页封面登录、同学账号登录、首次登录改密、全局 reveal 回收、基础性能测试等核心能力已经落地，并通过 `pnpm verify:all`。

但与上一版设计规格和实施计划相比，仍有明显未闭环的问题：

- 前言、人物长廊、相册、时间轴、年度册等页面仍有旧的 `section` 页面骨架，没有统一接入 `page-shell` / `page-header` / `paper-panel`。
- 多个组件仍残留 `museum-paper`、`color-museum-*`、`shadow-museum-*`，导致视觉语言在“纪念馆玻璃感”和“复古纸质同学录”之间摇摆。
- 首页未登录导航仍偏完整站点导航，没有完全做到“只服务登录入口”的克制状态。
- 首页 CTA 使用锚点跳转，但“平滑进入登录信息区”的交互约束还不够明确。
- 个人主页、照片墙仍使用 GSAP / ScrollTrigger，虽然测试限制了首页和部分页面初始加载，但动画所有权还没有形成全站级规范。
- 手机端和桌面端仍有一些页面只是压缩同一套结构，缺少各自的信息层级。
- 同学录的趣味功能已经存在，但图谱、座位记忆、留言贴纸、时间胶囊等亮点还没有被整合成清晰的站内体验。

本规格定义下一阶段的公开站点大幅重构方向。它既要收掉上一阶段遗留问题，也要为后续外观、动效、移动端体验和互动功能升级建立新的产品级基准。

## 2. 目标

1. 将公开站点统一升级为“复古纸质同学录 · 青春档案馆”的完整体验，而不是局部纸张化。
2. 完成上一阶段未闭环的页面迁移，消除核心页面中的旧 `museum-*` 视觉残留。
3. 重新定义桌面端和手机端的信息架构，让两端拥有各自适配的排版和操作节奏。
4. 把首页收敛为高质感登录入口，减少无关信息，强化“翻开同学录”的仪式感。
5. 将人物长廊、个人主页、相册、时间轴、年度册重构为互相统一但各有主题的纸质页面。
6. 让留言墙、图谱、座位记忆、时间胶囊等功能成为站内亮点，而不是零散插件。
7. 建立明确的动画所有权和性能约束，解决重复加载、跳变、卡顿和移动端压力。
8. 用静态测试、Playwright 网络测试、移动端视觉走查和 `verify:all` 锁定重构成果。

## 3. 非目标

- 不在本阶段重做 Worker 数据模型，除非公开站点功能必须补齐已有接口字段的展示。
- 不重做管理后台整体 UI；后台只在公开站点数据联动需要时做小范围兼容。
- 不引入重型 3D、全景引擎或大图纹理作为首屏核心。
- 不把首页做成站点目录页或营销页；首页仍只承担登录与入馆仪式。
- 不为了视觉装饰牺牲首屏速度、移动端滚动流畅度和可访问性。
- 不一次性重写全部组件逻辑；优先按页面和功能边界渐进迁移。

## 4. 产品定位

下一阶段的公开站点定位为：

> 一个可以被同学长期访问、补充资料、回看照片、留言互动的“纸质青春档案馆”。

它不是传统班级网站，也不是普通后台式信息表。用户进入后应感觉自己在翻阅一本带有档案袋、照片纸、手写便签、印章、信纸和纪念册页的同学录。整体应温暖、复古、高级、安静，同时保留轻量互动的趣味。

## 5. 总体体验架构

公开站点分为四个体验层：

1. **入馆层**
   - 首页封面。
   - 登录入口。
   - 首次登录密码引导。
   - 未登录导航的极简状态。

2. **浏览层**
   - 前言页。
   - 人物长廊。
   - 相册。
   - 时间轴。
   - 年度册。

3. **个人档案层**
   - 个人主页档案封面。
   - 基础资料。
   - 校园记忆。
   - 照片墙。
   - 留言墙。
   - 专属亮点模块。
   - 自助编辑入口。

4. **互动亮点层**
   - 班级关系图谱。
   - 座位记忆。
   - 时间胶囊。
   - 留言贴纸墙。
   - 资料完整度引导。

这四层需要共享同一套视觉系统、导航规则、动画生命周期和数据加载策略。

## 6. 视觉设计系统

### 6.1 核心风格

视觉关键词：

- 米白纸面。
- 旧书页边界。
- 档案袋。
- 信纸。
- 照片纸。
- 印章红。
- 棕色控件。
- 柔和金色细节。
- 低饱和便签。
- 细分隔线。
- 手账贴纸趣味。

站内页面应避免：

- 大面积玻璃拟态。
- 大面积纯黑背景，首页封面除外。
- 科技感蓝紫渐变。
- 过多漂浮装饰。
- 每个组件各自一套颜色和阴影。

### 6.2 全局页面骨架

所有公开站点常规页面必须统一接入以下语义：

- `page-shell`：页面背景、顶部留白、最小高度和纸张纹理。
- `page-header`：页面标题区，提供统一 label、标题、副文案。
- `content-grid`：内容主体的响应式网格。
- `paper-panel`：大型纸张面板。
- `paper-card`：普通纸张卡片。
- `archive-card`：人物或档案类卡片。
- `paper-section-divider`：纸页分隔线。

上一阶段中仍使用 `preface-page section`、`roster-page section`、`album-page section`、`timeline-page section` 的页面，需要全部迁移到统一骨架。

### 6.3 旧视觉残留清理

公开站点核心页面和核心卡片中，不应再依赖以下旧视觉锚点：

- `museum-paper`
- `color-museum-paper`
- `color-museum-ink`
- `color-museum-ink-soft`
- `shadow-museum-paper`

如需保留“纪念馆”概念，应通过文案和页面主题体现，而不是继续混用旧视觉 token。`museum-motion-soft` 也需要重新评估，避免与全局 reveal 重复。

### 6.4 组件风格

按钮：

- 主按钮使用纸棕色填充。
- 次按钮使用纸色背景和细边框。
- 危险/错误使用印章红。
- 移动端按钮最小高度不低于 44px。

输入框：

- 背景使用纸张白或柔和纸色。
- 聚焦态使用棕色描边和轻微阴影。
- 错误态要有文本提示，不只靠颜色。

卡片：

- 所有核心卡片使用纸面背景、细边框、轻阴影。
- 卡片圆角控制在 8-14px。
- 不把大页面 section 做成过度浮动的卡片墙。

图片：

- 头像、照片、相册图必须有固定尺寸或 `aspect-ratio`。
- 相册照片使用照片纸、拍立得或胶片风格，但统一边距和阴影。

贴纸：

- 留言和小标签可以保留贴纸趣味。
- 桌面端允许轻微错落和小角度旋转。
- 手机端默认不旋转，不 hover 放大，不造成横向抖动。

## 7. 页面级设计

### 7.1 首页与登录

首页保持“两段式登录体验”：

1. 暗色封面首屏。
2. 下滑后出现“翻开同学录，进入登录”按钮。
3. 点击按钮平滑滚动到登录表单。

优化要求：

- 未登录首页导航只展示品牌和必要入口，不展示完整站内导航。
- 封面不出现账号密码表单。
- 登录表单在桌面端为宽版纸张登记页，在手机端为单列纸页。
- 首次登录改密弹窗继续使用纸张样式。
- 首页不请求 `/api/classmates`，不加载图谱、座位记忆、GSAP 或 ScrollTrigger。
- 点击 CTA 的平滑滚动需要明确实现，不能只依赖浏览器默认锚点行为。

### 7.2 导航

桌面端：

- 登录后展示完整导航。
- 导航使用纸张胶囊或纸条样式。
- 当前页面状态清晰。
- 身份信息稳定显示，不闪烁。

手机端：

- 顶部保持轻量纸条。
- 菜单展开像纸页抽屉。
- 登录身份和退出入口放在菜单底部。
- 菜单打开后不造成页面横向滚动。

导航生命周期：

- 滚动监听必须有清理函数。
- Astro 页面切换后不得重复绑定。
- 登录态读取统一使用同学账号 session key。

### 7.3 前言页

目标：像翻开同学录后的第一页序章。

桌面端：

- 页面使用 `page-shell` 和 `page-header`。
- 正文放入窄纸页，最大宽度约 720px。
- 特别致谢和签名区使用纸张小卡片。

手机端：

- 单列信纸阅读。
- 行高略高。
- 操作按钮纵向排列，触控面积足够。

### 7.4 人物长廊

目标：从普通卡片网格升级为“同学档案检索墙”。

桌面端：

- 使用 `page-shell` 和 `page-header`。
- 搜索区改为档案夹条或检索纸条。
- 人物卡使用 `archive-card`，统一头像、姓名、座右铭、标签和资料状态。
- 排行榜、图谱、座位记忆改为同一纸质系统下的亮点卡。

手机端：

- 搜索区置于标题下方。
- 人物卡优先单列；如信息很少可以双列头像模式，但不能压缩座右铭。
- 排行榜和亮点模块使用折叠或横向分段卡。

必须清理：

- `RosterWall.vue` 中的 `museum-paper`。
- `ArchiveRosterCard.vue` 中的 `color-museum-*` 和 `shadow-museum-*`。
- `ClassGraphPreview.vue` / `SeatMapPreview.vue` 中的 `museum-paper`。

### 7.5 个人主页

目标：从长信息页升级为“个人档案袋”。

桌面端结构：

- 档案封面：头像、姓名、昵称、座右铭、资料完整度、标签。
- 左侧摘要：访问数、自助编辑、分享、资料完整度引导。
- 右侧内容：基础资料、联系方式、个性标签、兴趣馆藏、校园回忆、时间胶囊。
- 后续分区：照片墙、留言墙、班级图谱、座位记忆。

手机端结构：

- 头像、姓名、资料完整度优先。
- 各资料区单列折叠或短分区。
- 自助编辑入口放入内容流或底部安全区，不遮挡正文。
- 照片墙和留言墙有稳定占位，加载后不顶动页面。

动画要求：

- 个人主页不能同时让全局 reveal、Vue 动画、GSAP 作用于同一批元素。
- 照片墙入场动画可保留，但必须延迟到可见后加载，并且移动端降级。
- ScrollTrigger 只允许在确实必要的非首屏模块中按需加载。

### 7.6 相册页

目标：纸张相册，而不是普通图片网格。

桌面端：

- 使用 `page-shell` 和 `page-header`。
- 标签筛选为纸张分段控件。
- 相册分组使用纸页面板。
- 图片 3 列，保持稳定比例。

手机端：

- 图片 2 列。
- 标签筛选横向滚动。
- 灯箱按钮触控面积加大。
- 图片加载前预留尺寸。

### 7.7 时间轴

目标：纸页年表。

桌面端：

- 使用 `page-shell` 和 `page-header`。
- 保留左右交错或改为单侧年表均可，但线条、节点、卡片必须纸质化。
- 筛选 tab 改为纸张分段控件。

手机端：

- 强制单侧时间轴。
- 筛选 tab 横向滚动。
- 切换筛选时不发生大幅布局跳动。
- 图片保持固定比例。

### 7.8 年度册

目标：屏幕预览和打印导出都保持清晰。

屏幕端：

- 章节使用纸张面板。
- 封面、班级寄语、数据报告、同窗名录、青春瞬间、时光足迹、留言板保持统一章节样式。
- 去除不必要的 emoji 式标题装饰，改为印章、编号、细线或章节标签。

打印端：

- 保留干净白底打印规则。
- 不让深色首页和纸张纹理污染打印样式。
- 打印分页和内容层级不能被屏幕卡片样式破坏。

### 7.9 留言墙

目标：保留贴纸趣味，但让交互更稳。

桌面端：

- 贴纸样式可保留复古胶带、横格便签、拍立得、黑板等主题。
- 错落角度控制在小范围。
- 主人回复、表情反应和时间信息层级清晰。

手机端：

- 默认不旋转。
- 样式选择器横向滚动。
- 提交表单纵向排布。
- 禁止 hover 位移造成抖动。

### 7.10 互动亮点模块

本阶段可以增强，但必须作为非首屏亮点：

- 班级关系图谱：展示关系/共同标签/互动线索，默认懒加载。
- 座位记忆：以纸质座位表方式呈现，移动端可横向浏览。
- 时间胶囊：个人主页内的未来寄语或毕业留言展示。
- 资料完整度引导：帮助同学补齐头像、座右铭、照片、留言等资料。

亮点模块不应影响首页、前言页和人物长廊首屏性能。

## 8. 响应式规则

桌面端优先：

- 使用较宽容器展示档案、相册和年表。
- 允许两栏布局，但主阅读路径必须清楚。
- 页面 header 居中或左右结构均可，但同一页面内保持一致。

手机端优先：

- 每个核心页面必须有专门移动结构，而不是桌面栅格压缩。
- 常用操作进入拇指可触范围。
- 筛选、标签、菜单优先横向滚动或折叠。
- 不允许横向溢出。
- 不允许文字挤压、按钮文字溢出或卡片内容重叠。

断点建议：

- `<= 768px`：手机布局。
- `769px - 1023px`：窄平板布局。
- `>= 1024px`：桌面布局。

## 9. 动画与性能规范

### 9.1 动画所有权

每个页面只允许一个主入场系统：

- 首页：CSS 首屏入场 + CSS scroll reveal。
- 常规静态页：全局 reveal。
- 个人主页：页面主体优先全局 reveal，复杂模块按需局部动画。
- 照片墙：可见后加载局部动画；移动端降级。

禁止：

- 全局 reveal 与组件内部 GSAP 同时作用同一元素。
- 页面切换后重复注册 scroll listener。
- 每次数据刷新重新播放整页入场动画。
- 移动端使用重型 stagger、视差或 scrub。

### 9.2 加载稳定性

必须保证：

- 图片有稳定尺寸或 `aspect-ratio`。
- 懒加载模块有稳定 anchor 或 skeleton 高度。
- 首页不加载非登录必需 JS。
- 图谱、座位记忆、照片墙、留言墙均按需加载。
- SWR 或重新请求不能重置用户正在编辑的表单。

### 9.3 网络约束

首页首屏不得请求：

- `/api/classmates`
- 图谱模块脚本
- 座位记忆模块脚本
- GSAP
- ScrollTrigger

人物长廊首屏不得加载：

- 图谱实现脚本。
- 座位记忆实现脚本。
- GSAP / ScrollTrigger。

个人主页可以按需加载重模块，但必须通过 Playwright 网络测试验证加载时机。

## 10. 可访问性

- 所有按钮和输入框保留 `focus-visible`。
- 暗色首页文字对比度必须可读。
- 手机端触控目标不小于 44px。
- 登录错误、保存失败、加载失败必须有文本提示。
- 所有动效遵守 `prefers-reduced-motion`。
- 标题层级保持合理，不能为了视觉随意跳级。
- 交互元素不能只靠颜色表达状态。

## 11. 验收标准

### 11.1 视觉验收

- 首页首屏是暗色封面，不出现表单。
- 首页下滑后 CTA 渐显，点击后平滑进入登录区。
- 前言、人物长廊、相册、时间轴、年度册全部使用统一纸质页面骨架。
- 核心页面不再混用旧 `museum-paper` 风格。
- 个人主页像一份完整档案，而不是普通信息长表。
- 手机端没有横向滚动、按钮溢出、文字重叠。

### 11.2 功能验收

- 同学账号登录可完成。
- 首次登录改密可触发并完成。
- 页面切换后导航登录态稳定。
- 人物搜索、相册筛选、时间轴筛选可用。
- 个人主页照片墙、留言墙、图谱、座位记忆按需加载。
- 自助编辑入口不被重构破坏。

### 11.3 性能验收

- 首页不请求 `/api/classmates`。
- 首页不加载 GSAP / ScrollTrigger。
- 人物长廊首屏不加载图谱和座位记忆实现脚本。
- 页面切换后主内容不重复闪烁。
- 移动端滚动无明显卡顿。
- `pnpm verify:all` 通过。

### 11.4 代码验收

- 静态测试覆盖页面骨架迁移。
- 静态测试覆盖旧 `museum-*` 样式残留清理。
- Playwright 覆盖首页登录、首次登录、移动端无横向滚动、网络加载约束。
- 对视觉关键页保留手动 QA 截图或检查记录。

## 12. 建议实施阶段

### 阶段一：设计系统重建与旧样式清理

目标：

- 强化纸张 token。
- 增加通用布局和纸质组件 class。
- 为旧 `museum-*` 残留建立静态测试。
- 先不改业务逻辑。

### 阶段二：首页、登录和导航重构

目标：

- 优化首页封面和 CTA 平滑滚动。
- 首页未登录导航极简化。
- 登录表单和首次登录弹窗进一步统一纸张风格。
- 保持首页性能硬约束。

### 阶段三：核心浏览页面重构

目标：

- 重构前言、人物长廊、相册、时间轴、年度册。
- 补齐上一阶段未完成的 `page-shell` / `page-header` 迁移。
- 清理人物长廊相关旧视觉 token。

### 阶段四：个人主页深度重构

目标：

- 重排个人主页信息架构。
- 稳定照片墙和留言墙移动端体验。
- 将自助编辑、资料完整度、分享和互动模块纳入统一布局。
- 收敛 GSAP / ScrollTrigger 的使用边界。

### 阶段五：互动亮点增强

目标：

- 增强班级图谱、座位记忆、时间胶囊和留言互动。
- 保持非首屏、按需加载和移动端降级。
- 让亮点模块看起来属于同一个同学录系统。

### 阶段六：性能、视觉 QA 和发布收口

目标：

- 跑完整 `pnpm verify:all`。
- 增加 Playwright 网络与移动端测试。
- 做桌面端和手机端手动视觉走查。
- 清理无用样式和过时测试。
- 部署前确认工作区干净。

## 13. 风险与约束

- 视觉重构范围较大，必须按阶段提交，避免一次性大改后难以定位问题。
- 个人主页当前功能较多，重构时优先保持数据和交互不变，再调整结构。
- 清理 `museum-*` token 时要区分“旧视觉残留”和“仍被未迁移页面依赖的样式”，不能直接删除全局变量导致构建失败。
- GSAP / ScrollTrigger 不能简单粗暴移除，需要先判断是否影响照片墙和个人主页体验。
- 年度册同时服务屏幕预览和打印，屏幕纸张化不能破坏 `@media print`。
- 移动端视觉必须实测，不能只依赖静态断言。
