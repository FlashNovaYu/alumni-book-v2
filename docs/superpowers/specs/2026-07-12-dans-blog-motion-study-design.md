# Dan's Blog 动效调研与同学录迁移设计

日期：2026-07-12

## 背景与已确认决策

本次调研使用 Firecrawl 抓取 `https://danarnoux.com/` 的渲染 HTML、CSS 和品牌信息，并以浏览器核验首页、列表、个人页和长文页的实际表现。参考站使用 Astro View Transitions、预绘制主题状态、克制的浮层工具组、文字出入和圆形揭幕等技术，形成了细腻但不喧闹的阅读体验。

用户明确喜欢以下体验：

1. 导航切换时页面与文字的移动、渐变和进出节奏。
2. 明暗模式切换时以触发按钮为中心的全屏水波揭幕。
3. 整体的轻盈层次，而不是单一静态页面。

采用的方向是“渐进式档案动效”：保留同学录既有的纸张、档案、印章语言，提炼参考站的动效组织方式，不复制其技术博客的视觉主题或页面模板。

当前站点已有可复用基础：`MainLayout.astro` 已启用 `ClientRouter`；`global.css` 已有按前进/后退方向播放的页面过渡、班级信箱圆形过渡和 `prefers-reduced-motion` 降级；`navRuntime.ts` 已保存并延续导航活动标记位置。因此本设计以增强现有系统为原则，不引入第二套路由或动效框架。

## 调研结论

参考站中可借鉴的具体实现特征：

- 顶部导航的操作项位于低透明度、细边框、背景模糊的圆角工具组；悬停只上移 1px、略微提高背景和阴影强度。
- 导航活动状态使用连续位移的底标，而非每页重新出现的高亮块；移动菜单条目采用短暂错峰进入。
- 内容卡片有清晰的三层反馈：卡面、悬浮/阴影、标签或按钮胶囊。元信息、标题和摘要保持稳定层级。
- 标题与页面主体的入场节奏不同：正文承接页面切换，文字以短位移、透明度和去模糊完成二次进入；渐变只作一条暖色扫光，不承担内容可读性。
- 主题在 hydration 前从本地存储和系统偏好中确定，避免初始闪烁；切换时使用以按钮坐标为圆心的 `clip-path: circle()` 揭幕。
- 所有可选动效均尊重 `prefers-reduced-motion`。

## 范围与非目标

### 范围

- 将一级导航与页面转场整理为“档案翻页”动效：页面按导航实际方向平移、轻微透明度变化，并保留当前的 View Transition 架构。
- 为一级栏目之间的主标题新增共享元素转场：例如“同学档案”切到“班级空间”时，旧标题和新标题在同一视觉位置完成连续换位、淡变和渐变遮罩，而非两段互不相关的标题入场。
- 为标题、章节标题和有限数量的卡片添加编辑式文字入场：短距离上移、去模糊、暖金扫光；仅在页面初次进入时播放一次。
- 新增可持久化的“夜读模式”，采用深蓝墨、暖纸金和低饱和印章红；主题切换在支持 View Transitions 的浏览器中使用全屏圆形水波，不支持时无动画直接切换。
- 将导航工具区、档案卡、相册卡和班级空间卡的表面、标签和操作控件收敛为一致的三级反馈。
- 在内容足够长的学生档案页提供选择性锚点目录；短档案不得为了形式而展示空目录。
- 让档案卡逐步具备照片数、留言数、共同标签、最近更新等“可探索线索”，数据不足时不展示占位信息。

### 非目标

- 不复制参考站的终端状态栏、技术博客阅读量、字数、代码复制、评论登录或全站黑灰玻璃拟态。
- 不替换现有 Astro ClientRouter、信箱水波、卡片共享元素转场或身份鉴权流程。
- 不改变学生资料、相册、留言、私信和班级数据的服务端模型；探索线索只使用已有或后续明确增加的公开字段。
- 不将持续循环或大面积动画作为默认视觉；不在低性能设备或减少动态偏好下强制动画。
- 本设计文档不包含站点代码变更、生产部署或视觉资产替换。

## 方案比较

### 方案 A：仅增加夜读模式

实现最小，能带来水波效果，但无法回应用户最看重的导航文字和页面切换体验，也不会改善卡片体系。

### 方案 B：渐进式档案动效（采用）

在既有页面方向转场上增加纸页质感，在标题层补充短促文字入场，并把主题切换作为单独、可降级的圆形揭幕。卡片反馈同步统一，但不改动数据模型。这是与当前复古纸张设计令牌、导航实现和性能目标最兼容的方案。

### 方案 C：全面改造成玻璃拟态技术博客

能最接近参考站表面风格，但会稀释同学录的档案主题，并要求重做已有纸张令牌与大量页面。收益不足以覆盖重构和风格同质化风险。

## 设计细节

### 1. 页面切换：档案翻页

`MainLayout.astro` 的导航顺序必须从单独硬编码数组收敛到与 `TopNav.astro` 相同的一级导航顺序，确保“前进/后退”判断不会遗漏班级空间、年度册或更多页。进入目标页面时，新页面从相应水平位置进入；离开页面向反方向退场。位移保持轻量（约 24–40px）、时长约 280–360ms，缓动沿用现有 `cubic-bezier(0.22, 1, 0.36, 1)`。

纸张感来自暖色卡面、轻微透明度和导航墨线的伸缩，不使用 3D 翻书或页面卷曲。原因是后两种会与照片、表单和共享元素转场冲突，并降低可读性。

### 2. 跨页面标题变形

这是用户特别指定的 Blog ↔ Tags 式体验：页面主标题在路由切换中被视为同一个视觉锚点，而不是旧标题消失后再出现新标题。一级栏目页面的唯一 `<h1>` 使用稳定的共享 View Transition 名称（例如 `page-heading`），使浏览器在旧标题快照与新标题快照之间维持相同的几何位置；旧文字沿导航反方向轻移、降低透明度，新文字从相反方向轻移并渐显。两帧交替期间叠加一层短促的暖金到透明渐变遮罩，形成文字“换写”而不是硬切。

共享标题只适用于一级栏目或确实承担同一页面标题角色的页面。学生详情、搜索结果、聊天和用户生成标题不复用 `page-heading`，以免多个同名共享元素互相抢占，或与既有 `active-card` 详情页转场冲突。浏览器不支持 View Transitions 时，回退为下一节的一次性标题入场；减少动态偏好时直接显示新标题。

### 3. 文字入场与暖金扫光

仅对 `.page-header` 中的标签、主标题、简介，以及学生档案的主名称等稳定语义元素定义一次性入场。建议顺序为标签 → 标题 → 简介，间隔 50–90ms；文字移动不超过 12px，模糊只在初始帧存在且不超过 4px。暖金扫光是标题下方或标题内部的低不透明度渐变层，时长不超过 700ms。

不对正文段落、检索结果、聊天消息和用户输入内容做逐字动效，防止阅读干扰和列表重排。页面由浏览器后退缓存恢复时不重复播放。

### 4. 夜读模式与水波揭幕

主题状态以 `data-theme="paper" | "night"` 放在 `<html>` 上，并用单一存储键持久化。首次状态判定必须在首帧样式前的内联脚本完成：优先用户已选主题，否则使用系统深浅偏好。同步更新 `meta[name="theme-color"]`。

夜读模式只覆盖设计令牌，不创建与白天模式平行的组件 CSS：纸底转为深蓝墨，正文保留暖灰，金色和印章红降低饱和度，边框提高可辨识度。图片、头像和用户照片不做滤镜反相。

点击主题按钮时，若浏览器支持 `document.startViewTransition` 且用户未启用减少动态偏好，先记录按钮中心坐标，再在新根伪元素上播放 `clip-path: circle(0 at x y)` 到覆盖视口所需半径的圆形动画。若不支持或减少动态偏好，立即切换主题。该效果与既有 `vt-mailbox` 入口隔离：主题状态使用独立数据属性与专用伪元素规则，不复用或覆盖信箱转场。

### 5. 导航与卡片表面

保留当前 `TopNav.astro` 的纸张书签和印章红墨线。可在右侧账号、信箱和新增主题按钮之外套一层轻量工具组：半透明纸色、1px 柔和边框、背景模糊和极弱阴影。按钮悬停最多上移 1px、按下缩放至约 0.97；移动抽屉条目仍可错峰进入，但所有动画不超过 320ms。

人物档案、相册和班级空间共享三层结构：

1. 纸质卡面负责内容分组和常态边框。
2. 可点击卡在悬停/键盘聚焦时提高边框对比、上移 2–4px、增强阴影。
3. 标签、计数、筛选和低优先级操作采用紧凑胶囊，避免与标题争夺视觉焦点。

“照片数、留言数、共同标签、最近更新”是渐进信息：只显示真实、已获授权且不会暴露隐私的数据。数据不完整时宁可省略，也不显示“0”或虚构标签。

### 6. 长档案目录

学生资料在存在至少三个可导航的实质分区时，桌面端显示小型“档案目录”，锚定自我介绍、联系方式、影像、留言、音乐等已存在的页面区块；移动端并入已有抽屉或折叠控件。目录使用 IntersectionObserver 高亮当前位置。分区不足三项时不渲染目录。

## 可访问性、性能与回退

- `prefers-reduced-motion: reduce` 下，路由文字、菜单错峰、水波和卡片浮动均直接显示最终状态；主题立即切换。
- 主题按钮必须具有清晰的中文可访问名称、可见焦点样式和按下状态；不以颜色作为唯一状态提示。
- 所有 View Transition 规则均提供无 API 支持时的普通 DOM 跳转/主题切换回退。
- 不为效果引入大型动画库；使用已有 Astro View Transitions、CSS keyframes、`requestAnimationFrame` 和少量原生 DOM API。
- 不给每张卡片绑定持续 `mousemove`、无限动画或昂贵滤镜；列表和图片墙仍以首屏渲染、滚动性能为先。

## 验收标准

1. 一级导航任意相邻、非相邻方向切换均按实际左右关系播放；页面首次加载、浏览器后退和减少动态偏好下不会出现错误方向或重复入场。
2. 任意两个一级栏目之间切换时，唯一主标题以 `page-heading` 共享元素完成连续换位与渐变遮罩，不出现两段不相干的标题闪现；学生详情和聊天页面不会错误取得该共享名称。
3. 非共享标题只在新页面正常进入时以标签、标题、简介的顺序完成一次短入场；正文和聊天内容不逐字动画。
4. 主题首次加载无可感知的白/黑闪烁；点击主题按钮时，支持的浏览器从按钮坐标触发全屏圆形揭幕，其他情况安全即时切换。
5. 夜读模式在导航、人物档案、相册、班级空间、信箱与表单中均有足够文字和焦点对比；用户照片不被反相或破坏。
6. 卡片悬停、键盘焦点和按压反馈一致，且小屏无横向溢出、无必须依赖 hover 的操作。
7. 在 `prefers-reduced-motion: reduce` 下所有新增动效均停用；既有信箱水波和共享元素卡片转场不回归。
8. 后续实现至少通过站点类型检查、现有导航/转场 Playwright 用例和新增主题偏好、共享标题、动效降级的浏览器用例。
