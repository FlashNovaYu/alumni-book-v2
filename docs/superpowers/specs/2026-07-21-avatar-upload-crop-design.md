# 头像上传圆形裁切设计

## 背景与目标

阿里云自托管版本的公开站点允许同学在 `SelfEditPanel.vue` 中上传头像和背景图。当前选择头像后会直接压缩并上传，用户无法在上传前确认人物位置，导致非正方形照片在页面圆形头像中容易被错误裁切。本次改动只增强头像上传流程；背景图继续沿用现有直接上传流程。

目标：

- 选择头像后先打开客户端裁切预览，不立即上传。
- 使用固定正方形工作区和圆形选区，圆形内的内容作为最终头像。
- 支持拖拽调整位置和滑块缩放，移动端可用触摸指针操作。
- 用户确认后生成正方形图片文件，再复用现有压缩、响应式变体和 `/api/classmate/upload` 上传链路。
- 取消或裁切失败时不改变当前已保存头像；背景图行为不变。

## 方案与取舍

采用站点内置 Vue + Canvas 实现，不引入第三方裁切库。

- 优点：构建产物和阿里云运行时不增加依赖，能直接复用现有图片压缩/变体工具，裁切结果可单元测试。
- 代价：需要维护少量拖拽、缩放和资源释放逻辑。
- 不采用“自动居中裁切”：无法处理人物偏移，达不到用户可预览调整的目标。
- 不采用第三方库：当前需求只需要单一圆形选区，新增依赖的体积和 SSR/构建兼容风险高于收益。

## 交互设计

1. 用户点击头像“上传”并选择有效图片后，打开“调整头像”模态框；背景图输入仍直接调用原上传函数。
2. 模态框显示一个约 280px 的正方形预览区：图片铺满工作区，圆形描边表示选区，选区外使用半透明遮罩。下方提供“缩放”滑块和简短操作提示。
3. 图片初始按 cover 规则居中，缩放范围为 1–3 倍；拖拽时限制图片边缘不能露出选区。键盘可通过滑块调整缩放，按钮支持 Tab/Enter/Escape。
4. 点击“取消”关闭预览并释放本地对象 URL，不发起请求；点击“确认并上传”后进入现有上传状态，按钮禁用，成功后更新 `form.avatarUrl`。
5. 图片无法读取、Canvas 不可用或编码失败时显示中文错误消息并保留原头像。选择器在成功/失败/取消后清空，允许重新选择同一文件。

## 组件与数据流

- `SelfEditPanel.vue` 增加裁切状态：待处理文件、本地预览 URL、自然尺寸、缩放值、偏移量、拖拽指针状态和模态框显示状态。
- 预览层使用 `<img>` 的平移/缩放样式渲染，圆形选区由 CSS 遮罩和边框表达；不把圆形遮罩直接写入上传文件。
- `packages/shared/src/imageUtils.ts` 增加可测试的正方形裁切函数：接收源文件和源图像裁切矩形，使用 Canvas 输出 512×512 的 JPEG/PNG `File`。函数负责对象 URL/解码资源和 Canvas 异常清理。
- 头像确认流程调用裁切函数，再调用现有 `compressImage`、`generateImageVariants`、`appendImageVariants`，沿用原有字段、目录和鉴权头。背景图分支保持现有代码路径。

## 错误处理与兼容性

- 只接受现有 `accept="image/*"` 范围；GIF/SVG 等无法安全裁切的格式显示“该图片暂不支持裁切，请选择 JPG 或 PNG”，不触发上传。
- 使用 Pointer Events 统一鼠标和触摸拖拽；窗口失焦或指针结束时清理拖拽状态。
- 通过 `URL.revokeObjectURL`、`ImageBitmap.close`（若使用）和 Canvas 尺寸重置避免大图长期占用内存。
- API 401 继续交给现有 `handleClassmateUnauthorized`，网络/接口错误继续显示“上传失败”。

## 测试与验收

- 共享图片工具单元测试：横图/竖图裁切矩形映射到 512×512；Canvas 编码失败会抛出可识别错误；资源 URL 被释放。
- 站点静态测试：头像选择打开裁切流程，背景图仍调用原上传函数，确认按钮绑定上传，圆形选区和缩放控件具有中文可访问名称。
- 现有站点及共享包测试、类型检查和构建通过。
- 手动验收：在阿里云同源地址登录同学账号，选择一张偏移人物照片，拖拽/缩放后确认，页面头像与圆形选区一致；取消不会改变原头像；背景图上传仍成功。
