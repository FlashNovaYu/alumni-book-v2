# 界面反馈修复设计

## 目标

在保持现有纸质档案视觉语言和人物长廊入场动画的前提下，处理 7 条已确认的界面与交互反馈，并巡检公开站点主要页面的对齐和非预期倾斜。

## 范围与方案

### 人物长廊

保留页面和标题的现有淡入、位移动画。仅收敛首屏内容的宽度、网格间距与卡片基线，使搜索区和三列档案卡片在桌面端视觉上保持水平、稳定的对齐关系。

### 全站布局巡检

覆盖首页、前言、人物长廊、班级空间、年度册、影像馆、时间轴、留言页、账号中心与个人页。检查页面容器中心线、标题与内容区基线、网格列宽、水平溢出和非预期的 `rotate` / `skew` 变换。

目前代码扫描未发现全局页面容器的旋转或倾斜。首页纸张装饰、登录页 PASS 印章及个人页祝福贴纸墙含有刻意旋转：前两者保留；贴纸墙会作为视觉巡检项，仅在它破坏整体对齐时收敛为不旋转的卡片布局。

### 班级信箱头像

会话列表将相对头像路径通过 API 基础地址解析为可访问 URL。图片加载失败时，隐藏失效图片并展示姓名首字占位，避免破图图标。

### 年度册

不修改数据源和信息内容。页面以单一居中的纪念册纸张作为内容容器，统一封面信息、寄语、数据摘要、名录、影像与时间轴的标题层级、留白和边框；打印按钮独立于纸张内容。打印样式维持可分页、无导航按钮的输出。

### 顶部导航

品牌书本标识移除外框和底色，仅显示放大后的红色 SVG 书本图标；文字和导航布局不变。

### 班级群聊

聊天记录区域继续为独立可滚动容器；容器聚焦时支持滚轮和触控纵向滚动。向上滚动至顶部时自动请求更早的消息，同时保留“载入更早消息”作为可访问性与兜底入口，并在加载后维持原先的阅读位置。

### 精选影像

相册卡片改为完整照片封面：图片覆盖整张卡片，标题叠加在左上角，照片数量保留在右下角；标签区不再占用额外高度。没有封面时继续提供文本占位。

### 班级大事

时间轴按日期从新到旧排序，因此日期较早的记录显示在更下方；同日期保持既有 ID 稳定排序。

## 验证

1. 为相对头像 URL 解析、失效头像回退、时间轴倒序与相册覆盖层结构补充或调整针对性测试。
2. 运行相关组件/工具测试，并执行公开站点构建。
3. 通过本地预览检查桌面与窄屏下的聊天滚动、相册覆盖文字和年度册打印布局。
4. 对上述主要页面进行桌面与窄屏布局巡检，确保主要内容容器无非预期旋转、无水平溢出，且居中区块的中心偏差不超过 1 像素。

## 非目标

- 不改变 API 协议、数据库字段或已有内容数据。
- 不重做全站主题，不新增动画库或图片资源。
